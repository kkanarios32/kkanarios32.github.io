Vector Processing Unit [OGCRN]

  • May 22, 2026
  • Kellen Kanarios

A core component of the TPU architecture responsible for elementwise operations.

  • 2D SIMD execution, where each of 1024 sites execute same program in lockstep.
  • Each site contains 4 ALUs, enabling 4096 FLOPs.
  • Fed a 8x128 Vector Register in one cycle.

    Matrix Multiply Unit (MXU) [OGCRP]

    • May 22, 2026
    • Kellen Kanarios

    A core component of the TPU architecture responsible for performing matrix multiplication via a systolic array.

    • 256x256 totaling 65,536 multiply accumulate units i.e. each unit can multiply and add in one cycle.
      • Remark. Can only do 256x256 matrix multiplication, so the order is
      1. Tiling the matrix to load from HBM to VMEM
      2. Compiler tiles from VMEM to 256x256 tiles to load into MXU.

      Vector ISA / SIMD [Q7ZD]

      • May 12, 2026

      Single instruction, multiple data. Extend the Fetch-execute loop so that one fetch+decode applies the same operation to a vector of operands. The fetch-decode overhead amortizes over the vector width; loop bookkeeping (increment, compare, branch) is replaced by a single vector op.

      Concretely, a vector ISA adds:

      • A vector register file — wider than scalar registers; specify the data width per element and total bits per register.
      • Vector load/store instructions for moving contiguous chunks of memory to/from vector registers efficiently.
      • Vector predication (masking) — apply the op only where the mask is true, allowing data-dependent control flow without scalar branches.

      Trade-offs:

      • Need the array length to be divisible by the vector width for perfect speedup; tail handling is annoying.
      • Vector pipelines cannot do Forwarding cheaply — would need \(O(n^2)\) connections to every ALU.
      • Wider hardware vectors can be supported on narrower hardware by microcoded decomposition; backward compatibility falls out for free.

      In practice, vector code is generated by:

      1. Intrinsic functions (architecture-specific C builtins).
      2. Vector data types in a language extension.
      3. External libraries (BLAS, etc.).
      4. Compiler auto-vectorization.
      Problem. Aliasing: modifying a vector you're also reading from causes a race within the operation.
      Remark. SIMD is hardware-driven data parallelism. The GPU descendant is SIMT, which softens many of SIMD's rigidities by giving each lane its own program counter and active mask.
