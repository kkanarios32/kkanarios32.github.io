1.1. POTW: Multi-Level Discovery of Deep Options [ZUAG]

  • July 22, 2026
  • Kellen Kanarios

The setup for Multi-Level Discovery of Deep Options:

  • We have a meta-controller that outputs the skill \(z\) given the state \(s\) i.e. \(z_t \sim \eta (\cdot \mid s_t)\)
  • We have the skill, which is a skill-conditioned policy \(a_t \sim \pi (\cdot \mid s_t, z_t)\)
  • We have the termination function \(\psi (\cdot \mid s_t, z_{t - 1})\), which is per skill and evaluates the previous skill \(z_{t - 1}\) at the new state \(s_t\), such that \(b_t \sim \mathrm {Ber}\big (\psi (s_t, z_{t - 1})\big ) \in \{0, 1\} \)
    • For \(b_t = 1\), we resample a new skill \(z_t \sim \eta (\cdot \mid s_t)\)
    • Otherwise, \(z_{t} = z_{t - 1}\)

Assuming there is a true underlying latent history \(\xi = \{b_{0}, z_{0}, b_{1}, z_{1}, \ldots \} \) and the observable history \(\mathcal {H} = \{s_{0}, a_{0}, s_{1}, a_{1}, \ldots \} \), the likelihood according to the above generative model is given by

\[\mathbb {P}(\mathcal {H}, \xi ) = p(s_{0})\delta _{\{b_{0} = 1\}}\eta (z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi (a_t \mid s_t, z_t) p(s_{t+1} \mid s_t, a_t)\]

Furthermore, according to our model, we have that

\[ \begin {align*} \mathbb {P}(b_t = 0, z_t \mid z_{t - 1}, s_t) &= (1 - \psi (s_t, z_{t - 1})) \cdot \delta _{\{z_{t} = z_{t - 1}\}} \\ \mathbb {P}(b_t = 1, z_t \mid z_{t - 1}, s_t) &= \psi (s_t, z_{t - 1}) \cdot \eta (z_t \mid s_t) .\end {align*} \]

Up to this point, we have assumed this is the ground truth model and we are given each of the components. Instead we want to learn these components. To do so, we introduce our set of parameters \(\mathbf {\theta } \), such that each component is parametrized by a subset of \(\mathbf {\theta } \) i.e. \(\eta _{\theta }, \pi _{\theta }, \psi _{\theta }\).

Goal. We want to learn some \(\theta \) that maximized the likelihood of the observable data.

To do so, first consider the log-likelihood of our new parametrized model

\[ \begin {align*} \log \mathbb {P}_{\theta }(\mathcal {H}, \xi ) &= \log \left [p(s_{0})\delta _{\{b_{0} = 1\}}\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t) p(s_{t+1} \mid s_t, a_t)\right ] \\ &= \log p(s_{0})\delta _{\{b_{0} = 1\}} + \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t) \right ] + \sum _{t=0}^{T-1} \log \left [p(s_{t+1} \mid s_t, a_t)\right ] \\ &= \underbrace {\log p(s_{0})\delta _{\{b_{0} = 1\}} + \sum _{t=0}^{T-1} \log \left [p(s_{t+1} \mid s_t, a_t)\right ]}_{\text {independent of } \theta } + \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t)\right ] .\end {align*} \]

Since we are only concerned with learning \(\theta \), we can disregard the \(\theta \) independent terms leaving

\[\begin {align*} \nabla _{\theta } \log \mathbb {P}_{\theta }(\mathcal {H}, \xi ) &= \nabla _{\theta } \left [ \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t)\right ] \right ]\\ &= \nabla _{\theta } \log \left [\eta _{\theta }(z_{0} \mid s_{0})\right ] + \sum _{t=1}^{T - 1} \nabla _{\theta } \log \left [\mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t)\right ] + \sum _{t = 0}^{T - 1} \nabla _{\theta } \log \left [\pi _{\theta }(a_t \mid s_t, z_t)\right ]. \end {align*}\]
Problem. This requires samples \((\mathcal {H}, \xi ) \sim P_{\theta }(\mathcal {H}, \xi )\), but we only have samples \(\mathcal {H} \sim P_{\theta }(\mathcal {H})\)!!!
Solution. We can use the trick from Expectation Maximization. Referred to in the paper as the expectation gradient trick. Namely, \[\nabla _{\theta } \log P_{\theta }(\mathcal {H}) = \mathbb {E}_{\xi \sim P_{\theta }(\xi \mid \mathcal {H})} \left [ \nabla _{\theta } \log P_{\theta }(\mathcal {H}, \xi ) \right ]\]

Now it remains to compute the expectation from \(P_{\theta }(\xi \mid \mathcal {H})\)

Remark. Note that \(P_{\theta }(\xi \mid \mathcal {H})\) is a distribution over \((K, \{0,1\})^{T} \) i.e. over each \((z_t, b_t)\). To efficiently compute, we need to use Forward-backward algorithm.

Computing the expectation

For,

\[\begin {aligned} u_t(z) &= \mathbb {P}_{\theta }(z_t = z | \xi ) \\ v_t(z) &= \mathbb {P}_{\theta }(b_t = 1, z_t = z | \xi ) \\ w_t(z) &= \mathbb {P}_{\theta }(z_t = z, b_{t+1} = 0 | \xi ). \end {aligned}\]

We have that

\[\nabla _{\theta } L[\theta ; \xi ] = \sum _{h \in \mathcal {H}} \left ( \sum _{t=0}^{T-1} \left ( v_t(z) \nabla _{\theta } \log \eta (z|s_t) + u_t(z) \nabla _{\theta } \log \pi _h(a_t|s_t) \right ) + \sum _{t=0}^{T-2} \left ( (u_t(z) - w_t(z)) \nabla _{\theta } \log \psi _h(s_{t+1}) + w_t(z) \nabla _{\theta } \log (1 - \psi _h(s_{t+1})) \right ) \right ).\]

Following the Forward-backward algorithm, we can get the posterior for the latent i.e. \(\mathbb {P}((z_{t}, b_{t}) \mid \mathcal {H})\). We have the corresponding forward and backward components

\[\begin {align*} \phi _t(z, b) &= \mathbb {P}(s_{0}, a_{0}, \ldots , s_{t}, z_{t} = z, b_{t} = b) \\ \beta _t(z, b) &= \mathbb {P}(s_{t + 1}, a_{t + 1}, \ldots , s_{T} \mid z_{t} = z, b_{t} = b), .\end {align*}\]

which can then be written as

\[\begin {align*} \phi _t(z, b) &= \mathbb {P}(s_t, a_t \mid z, b) \sum _{(z', b')} \phi _{t - 1}(z', b') \mathbb {P}(z, b \mid z', b') \\ \beta _t(z, b) &= \sum _{(z', b')} \beta _{t + 1}(z', b') \mathbb {P}(s_{t + 1}, a_{t + 1} \mid z', b') \mathbb {P}(z', b' \mid z, b) .\end {align*}\]

Forward pass. For the forward pass, we have the initial condition

\[\phi _0(z, b) = p_{0}(s_{0}) \delta _{\{b = 1\}} \eta (z \mid s_{0})\]

Here \((z', b')\) ranges over the possible values of \((z_{t - 1}, b_{t - 1})\). It is sufficient to expand the remaining two terms in terms of our generative model as defined above. First, note that \((z_{t}, b_{t})\) are independent of \(b_{t - 1}\). Therefore, we can drop the \(b_{t - 1}\) term and are left with the following two cases:

\[\begin {align*} \mathbb {P}_{\theta }(z, b = 0 \mid z') &= \delta _{\{z = z'\}} (1 - \psi _{\theta }(s_{t}, z')) \\ \mathbb {P}_{\theta }(z, b = 1 \mid z') &= \eta _{\theta }(z \mid s_{t}) \psi _{\theta }(s_{t}, z') .\end {align*}\]

Plugging this in, we now have

\[\begin {align*} \phi _t(z, b) = \mathbb {P}_{\theta }(s_t, a_t \mid z, b) \sum _{(z', b')} \phi _{t - 1}(z', b') \Big (&\eta _{\theta }(z \mid s_{t}) \psi _{\theta }(s_{t}, z') \\ &+ \delta _{\{z = z'\}} (1 - \psi _{\theta }(s_{t}, z'))\Big ) .\end {align*}\]

It remains to find \(\mathbb {P}_{\theta }(s_t, a_t \mid z, b)\). Once again, given \(z_t\) we no longer care about \(b_t\). Therefore, this becomes

\[\begin {align*} \mathbb {P}_{\theta }(s_t, a_t \mid z) &= \pi _{\theta }(a_t \mid s_t, z) p(s_t \mid s_{t - 1}, a_{t - 1}) .\end {align*}\]

This gives us our final expression

\[\begin {align*} \phi _t(z, b) = &\pi _{\theta }(a_t \mid s_t, z) p(s_t \mid s_{t - 1}, a_{t - 1}) \\ &\times \sum _{(z', b')} \phi _{t - 1}(z', b') \Big (\eta _{\theta }(z \mid s_{t}) \psi _{\theta }(s_{t}, z') + \delta _{\{z = z'\}} (1 - \psi _{\theta }(s_{t}, z'))\Big ) .\end {align*}\]
Remark. We have abused notation here, allowing \(\mathbb {P}(s_t, a_t \mid z_t, b_t)\) to depend on \(s_{t - 1}, a_{t - 1}\), etc. This is to align with the notation from Forward-backward algorithm. It makes no difference in practice as long as the backward dependence is finite / bounded. However, in practice you would want the history dependence to be small and here it is only one-step.

Backward pass. For the backward pass, we define the end term as

\[\beta _{T-1}(z, b) = \pi (a_{T - 1} \mid z, s_{T - 1}) p(s_{T} \mid s_{T - 1}, a_{T - 1})\]

Here \((z', b')\) ranges over the possible values of \((z_{t + 1}, b_{t + 1})\). The other terms are analagous i.e.

\[\begin {align*} \mathbb {P}_{\theta }(z', b' = 0 \mid z) &= \delta _{\{z' = z\}} (1 - \psi _{\theta }(s_{t + 1}, z)) \\ \mathbb {P}_{\theta }(z', b' = 1 \mid z) &= \eta _{\theta }(z' \mid s_{t + 1}) \psi _{\theta }(s_{t + 1}, z) .\end {align*}\]

and

\[\begin {align*} \mathbb {P}_{\theta }(s_{t + 1}, a_{t + 1} \mid z') &= \pi _{\theta }(a_{t + 1} \mid s_{t + 1}, z') p(s_{t + 1} \mid s_t, a_t) .\end {align*}\]

1.2. POTW: Learning Options via Compression [G27U]

  • July 23, 2026
  • Kellen Kanarios

The setup for Learning Options via Compression.

  • A state abstraction posterior \(q_{\theta }(s_{t} \mid z_{t}, x_{t})\) and decoder \(p_{\theta }(a_{t} \mid s_{t})\), where \(\pi _{\theta }(a_t \mid s_t, x_t) = \mathbb {E}_{q_{\theta }(s_{t} \mid x_{t}, z_{t})}[p_{\theta }(a_t \mid s_t)]\)
    • This seems to be needed to ensure that the decoder cannot overfit to the offline data?
  • A termination network \(\beta _{\theta }(b_t \mid x_{1}, \ldots , x_{t})\)
  • A skill posterior \(\eta _{\theta }(z_{t} \mid z_{t - 1}, b_{t}, x_{1:T},a_{1:T})\).

Unlike in DDO, where they do exact posterior inference, here we do variational inference. Namely, they directly try to model the posterior via the neural network \(\eta _{\theta }(z_{t} \mid z_{t - 1}, b_{t}, x_{1:T},a_{1:T})\). This has a few consequences:

  • The expectation gradient is now true Expectation Maximization. In particular, we need to optimize the ELBO.

The ELBO of LOVE can be written as

\[\begin {align*} \mathcal {L}_{\text {ELBO}}(\theta ) \approx \sum _{t=1}^{T} \log &p_{\theta }(\mathbf {a}_{t} \mid \mathbf {s}_{t}) + D_{\text {KL}}(q_{\theta }(m_{t} \mid \mathbf {x}_{1:t}) \parallel p_{\theta }(m_{t} \mid \mathbf {s}_{t})) \\ &+ D_{\text {KL}}(q_{\theta }(z_{t} \mid m_{t}, \mathbf {x}_{1:T}, \mathbf {a}_{1:T}) \parallel p(z_{t})) + D_{\text {KL}}(q_{\theta }(s_{t} \mid z_{t}, \mathbf {x}_{t}) \parallel p_{\theta }(s_{t} \mid s_{t-1}, m_{t-1})) .\end {align*}\]

Notably this requires a few additional prior networks for the KL regularization term in the ELBO.

  • A boundary prior \(\beta (b_t \mid s_{t})\).
  • A skill prior \(p(z)\).
  • A state abstraction transition prior \(p(s_t | s_{t-1}, z_t)\).

Additionally, we also need to sample the \(z\)'s. This is done via Gumbel softmax and causes much higher variance.

2. Frankenstein-ing LOVE and DDO

  • July 23, 2026
  • Kellen Kanarios

We find ourselves in-between LOVE and DDO. In particular, in DDO, they are concerned with learning a hierarchy of policies where the higher-level meta policy \(\eta \) must be usable online only given the current state.

  • (+) The learned termination conditions \(\beta \) are dependent on the active skill.
  • (+) Exact inference \(\implies \) lower variance.
  • (-) Weak skill model \(\eta (z \mid s)\).
  • (-) No explicit encouragement for temporally-extended skills.

On the contrary, in LOVE, they treat skills as a pure sequence-modeling problem. The posterior \(q_{\theta }(z_{t} \mid z_{t - 1}, b_{t}, x_{0}, \ldots , x_{T})\) is purely used to label offline trajectories and give the skills meaning. To make them usable online, it is then required to learn an additional hierarchical policy over these skills.

  • (+) The skill-labeler is allowed to use not only the current state but also the future visited states.
  • (+) Compression objective encourages temporally extended skills.
  • (-) Design inherently does not allow the terminations to be skill-dependent.
  • (-) Need to sample \(z\)'s to compute gradients \(\implies \) higher-variance.

Our graphical model. We try to combine the desirable properties.

We essentially combine the insight of LOVE where instead of trying to model the full joint distribution i.e. \(\mathbb {P}(\mathcal {H}, \xi )\) we instead model \(\mathbb {P}(\mathcal {A}, \xi \mid \mathcal {S})\), where we treat the entire state history as known.

In the language of HMMs, the emission probabilities only correspond to the actions. Therefore, we can perform the Forward-backward algorithm and condition on whatever component of the state we want without breaking the exactness.

Remark. This is only a good idea for us because we do not need to use the meta controller during evaluation.

Deliberation cost. In LOVE, they encourage temporal extension through the compression loss. This requires an empirical approximation of the skillmarginal \(p(z)\). For me, this caused the skill entropy to just collapse to zero instead of encouraging temporally extended options. Instead, we use a deliberation cost more common in the options literature The Option-Critic Architecture, where we just add the switch-rate as an additional loss term.

  • We are given the posterior \(\mathbb {P}(z_t = z, b_t = 1 \mid \mathcal {H})\). In our implementation this would be \((B, T, K)\)
  • We marginalize over skills \(z\) by summing to get \(\mathbb {P}(b_t = 1 \mid \mathcal {H})\)
  • Then average over batch \(B\) and time \(T\) to get expected reads.

As in LOVE, we need to balance minimizing log-likelihood with compression / switching cost. To do so, we do dual descent.

  • First, without compression, we run the algorithm and observe what is approximately the best attainable log-likelihood loss is.
  • Then, we minimize the deliberation cost subject to \(\mathcal {L}_{\text {likelihood}} \leq C + \epsilon \)

3. Implementation details

  • July 23, 2026
  • Kellen Kanarios
  • The skill network \(\eta (z_t \mid s_{0}, \ldots s_{T})\) is a bidirectional GRU, where the hidden states are accumulated via running the GRU forwards and backwards and concatenating the results.
  • The termination network \(\beta (b_t \mid z_{t - 1}, s_{0}, \ldots , s_{t})\) is a GRU, that only runs forward.
    • Instead of passing the previous skill as input, the output dimension is size \(K\) (number of skills).
  • The skill policy network \(\pi _{\theta }(a \mid s, z)\) is a standard Gaussian policy that take \(z\) as a one-hot encoding.

4. Planning over learned skills

  • July 23, 2026
  • Kellen Kanarios

Setup. Assume we are given a set of skills \(\mathbf {z} \in \{0, \ldots , K\}\) along with a skill-conditioned lower-level policy \(\pi (s \mid z)\). Then what we want is to be able to learn the optimal goal-reaching policy over these skills. To do so, we must go beyond traditional offline skill-learning algorithms. We must Learn a value function over learned skills purely from offline data

Skill-conditioned GCIQL. If we now define the analagous skill-conditioned GCIQL losses, we have \[\begin {align} \mathcal {L}_{V}(\theta _{V}) &= \mathbb {E}_{(s_{t}, z_{t}, g) \sim \mathcal {D}} \Big [L_{2}^{\tau }\big (Q_{\overline {\theta }_{Q}}(s_{t},z_{t},g) - V_{\theta _{V}}(s_{t},g)\big )\Big ] \\ \mathcal {L}_{Q}(\theta _{Q}) &= \mathbb {E}_{(s_{t}, z_{t}, s_{t+\kappa }) \sim \mathcal {D}} \Big [\big (r(s_{t}, z_{t}, g) + \gamma V_{\overline {\theta }_{V}}(s_{t+\kappa }, g) - Q_{\theta _{Q}}(s_{t},z_{t},g) \big )^2\Big ] \end {align}\] This introduces a few changes and constraints on the final algorithm.

  • \(\kappa \) is a random variable dependent on the termination condition of \(z\).
  • It is not immediately obvious how we should design \(r(s_t, z_t, g)\).
Future states. For the first point, there are two options with trade-offs. In LOVE, they learn a model over trajectories that partitions them into unique skills. We can directly use this to get the intermediate \(s_{t + \kappa }\) just based off the termination output by the model.
  • This is the simplest approach and basically a generalization of HIQL to varying length skills, where they just use a state \(k\) steps away in the trajectory.
  • However, this approach would be limited on stitching datasets, where all of the trajectories are short.

The second approach, assuming we already have skills (given termination function), to perform better in the stitching setting is instead to learn a generative model over the state \(s_{t+\kappa }\). Then we could just use \(s_{t + \kappa } \sim p(s_{t+\kappa } \mid s_t, z_t)\) as our next state in \eqref{eq:gciql-qz}. There is new work on universal horizon models[chungOfflineReinforcementLearning2026] focused on this for \(n\)-step models. Alternatively, it seems there is work directly learning the value function for a sequence of skills [farebrotherCompositionalPlanningJumpy2026], but this seems to only work for a fixed (per skill) discount factor.

Reward. Here, I propose a modified reward function based on the observation that goal-conditioned policies work well for short-horizon goal-reaching. One of our skills is hardcoded to instead be a learned goal-conditioned policy \(\pi (a \mid s, g)\) with associated value function \(V(s, g)\). Denote this skill as \(z_g\). Then we can define the reward function as

\[\begin {align*} r(s, z, g) &= \begin {cases} 1, & \text {if } z = z_g \text { and } V(s, g) < \epsilon \\ 0, & \text {otherwise} \end {cases} .\end {align*} \]

Note that it may be the case that the high-level policy learns to just choose \(z_g\) everytime, so we may want to penalize that decision somehow for \((s,g)\) pairs with large \(V(s,g)\). Intuitively, this would end up with a higher-level policy that best utilizes the temporally-extended skills to reach the nearby region of \(g\) then concedes to the lower-level policy.

5. Outstanding Problems / Future directions

  • July 23, 2026
  • Kellen Kanarios

Extending beyond pointmaze in locomotion. The main issue faced here is that our discriminator is too powerful in high dimensional state space. Namely, we score options based on their ability to reconstruct actions i.e. if \(\mathbb {P}(a_t \mid s_t, z_t = z)\) is high then we further increase the probability of selecting that option \(z\). In higher-dimensional state spaces, the skill network \(\pi _{\theta }(a_t \cdot \mid s_t, z)\) is powerful enough to reconstruct \(a_t\) purely from \(s_t\) without using \(z\). This basically provides no signal for which \(z\) to use, and we end up just selecting \(z\) uniformly every time step.

  • I have seen a little sign of life from trying \(\pi (a \mid f_{\theta }(s), z)\) i.e. compressing \(s\) to a lower dimensional representation.
  • It may also be worth trying a really small network. Right now \((256, 256)\) already gets very low reconstruction loss, so maybe we can go smaller?

Conquering manipulation. In OGBench: Benchmarking Offline Goal-Conditioned RL, they provide a suite of manipulation environments in addition to the standard locomotion. One such environment, is the manipulation equivalent of the lights out game. This is a great test-bed to separate skill-learning from planning. In particular, the skill-learning portion can be clearly decomposed into the pressing of each of the buttons. However, when directly trying to use the LOVE it is not able to distinguish between these skills.

  • On puzzle 3x3 LOVE terminates every \(1.4\) steps because all the skills look similar.
  • This is using the threshold \(\beta > 0.9\).

Conjecture. 

  • July 23, 2026
  • Kellen Kanarios

Pressing each of the buttons looks too similar, where the main discrepancy between them is only their termination condition. Since LOVE's termination condition is independent of skill, it is unable to distinguish between these.

  • This is further confirmed in the scene environment. LOVE is able to distinguish between very different skills like open drawer vs. window. However, it is not able to distinguish between unlock drawer vs. window, where they only differ in pressing two adjacent buttons.

Injecting goals into the graphical model. Another failure mode of manipulation is that skills cannot be goal-independent. For example, in the generating policy of scene they have a pick and place skill that picks up the block and places it somewhere. Where to place the block is encoded in the goal. We can augment our graphical model to also allow the actions to depend on the goal.

However, we do not want the skills to become goal-conditioned policies. To get around this we can use the information bottleneck idea from InfoBot: Transfer and Exploration via the Information Bottleneck. Namely, we penalize the Mutual Information between the goal and action or goal and skill (TBD the right way to do this).

Remark. Ideally, this would allow goal-dependent skills, where the goal is not the entire state space. For example, penalizing Mutual Information could cause just the desired block position as the information to be passed to the skill. This ignores the irrelavant components of the state space for that skill. Another skill may focus on whether the window is locked / unlocked, etc.

Get rid of weird threshold / turn into MPC-ish. Currently, when planning, we must tune the value_threshold for when we pass control to goal-conditioned policy. To get 89 percent on pointmaze we used -10. Instead of using a threshold, we could instead just treat the goal-conditioned policy as another skill, but the future state is just .

        POTW: Multi-Level Discovery of Deep Options [ZUAG]

        • July 22, 2026
        • Kellen Kanarios

        The setup for Multi-Level Discovery of Deep Options:

        • We have a meta-controller that outputs the skill \(z\) given the state \(s\) i.e. \(z_t \sim \eta (\cdot \mid s_t)\)
        • We have the skill, which is a skill-conditioned policy \(a_t \sim \pi (\cdot \mid s_t, z_t)\)
        • We have the termination function \(\psi (\cdot \mid s_t, z_{t - 1})\), which is per skill and evaluates the previous skill \(z_{t - 1}\) at the new state \(s_t\), such that \(b_t \sim \mathrm {Ber}\big (\psi (s_t, z_{t - 1})\big ) \in \{0, 1\} \)
          • For \(b_t = 1\), we resample a new skill \(z_t \sim \eta (\cdot \mid s_t)\)
          • Otherwise, \(z_{t} = z_{t - 1}\)

        Assuming there is a true underlying latent history \(\xi = \{b_{0}, z_{0}, b_{1}, z_{1}, \ldots \} \) and the observable history \(\mathcal {H} = \{s_{0}, a_{0}, s_{1}, a_{1}, \ldots \} \), the likelihood according to the above generative model is given by

        \[\mathbb {P}(\mathcal {H}, \xi ) = p(s_{0})\delta _{\{b_{0} = 1\}}\eta (z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi (a_t \mid s_t, z_t) p(s_{t+1} \mid s_t, a_t)\]

        Furthermore, according to our model, we have that

        \[ \begin {align*} \mathbb {P}(b_t = 0, z_t \mid z_{t - 1}, s_t) &= (1 - \psi (s_t, z_{t - 1})) \cdot \delta _{\{z_{t} = z_{t - 1}\}} \\ \mathbb {P}(b_t = 1, z_t \mid z_{t - 1}, s_t) &= \psi (s_t, z_{t - 1}) \cdot \eta (z_t \mid s_t) .\end {align*} \]

        Up to this point, we have assumed this is the ground truth model and we are given each of the components. Instead we want to learn these components. To do so, we introduce our set of parameters \(\mathbf {\theta } \), such that each component is parametrized by a subset of \(\mathbf {\theta } \) i.e. \(\eta _{\theta }, \pi _{\theta }, \psi _{\theta }\).

        Goal. We want to learn some \(\theta \) that maximized the likelihood of the observable data.

        To do so, first consider the log-likelihood of our new parametrized model

        \[ \begin {align*} \log \mathbb {P}_{\theta }(\mathcal {H}, \xi ) &= \log \left [p(s_{0})\delta _{\{b_{0} = 1\}}\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t) p(s_{t+1} \mid s_t, a_t)\right ] \\ &= \log p(s_{0})\delta _{\{b_{0} = 1\}} + \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t) \right ] + \sum _{t=0}^{T-1} \log \left [p(s_{t+1} \mid s_t, a_t)\right ] \\ &= \underbrace {\log p(s_{0})\delta _{\{b_{0} = 1\}} + \sum _{t=0}^{T-1} \log \left [p(s_{t+1} \mid s_t, a_t)\right ]}_{\text {independent of } \theta } + \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t)\right ] .\end {align*} \]

        Since we are only concerned with learning \(\theta \), we can disregard the \(\theta \) independent terms leaving

        \[\begin {align*} \nabla _{\theta } \log \mathbb {P}_{\theta }(\mathcal {H}, \xi ) &= \nabla _{\theta } \left [ \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t)\right ] \right ]\\ &= \nabla _{\theta } \log \left [\eta _{\theta }(z_{0} \mid s_{0})\right ] + \sum _{t=1}^{T - 1} \nabla _{\theta } \log \left [\mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t)\right ] + \sum _{t = 0}^{T - 1} \nabla _{\theta } \log \left [\pi _{\theta }(a_t \mid s_t, z_t)\right ]. \end {align*}\]
        Problem. This requires samples \((\mathcal {H}, \xi ) \sim P_{\theta }(\mathcal {H}, \xi )\), but we only have samples \(\mathcal {H} \sim P_{\theta }(\mathcal {H})\)!!!
        Solution. We can use the trick from Expectation Maximization. Referred to in the paper as the expectation gradient trick. Namely, \[\nabla _{\theta } \log P_{\theta }(\mathcal {H}) = \mathbb {E}_{\xi \sim P_{\theta }(\xi \mid \mathcal {H})} \left [ \nabla _{\theta } \log P_{\theta }(\mathcal {H}, \xi ) \right ]\]

        Now it remains to compute the expectation from \(P_{\theta }(\xi \mid \mathcal {H})\)

        Remark. Note that \(P_{\theta }(\xi \mid \mathcal {H})\) is a distribution over \((K, \{0,1\})^{T} \) i.e. over each \((z_t, b_t)\). To efficiently compute, we need to use Forward-backward algorithm.

        For,

        \[\begin {aligned} u_t(z) &= \mathbb {P}_{\theta }(z_t = z | \xi ) \\ v_t(z) &= \mathbb {P}_{\theta }(b_t = 1, z_t = z | \xi ) \\ w_t(z) &= \mathbb {P}_{\theta }(z_t = z, b_{t+1} = 0 | \xi ). \end {aligned}\]

        We have that

        \[\nabla _{\theta } L[\theta ; \xi ] = \sum _{h \in \mathcal {H}} \left ( \sum _{t=0}^{T-1} \left ( v_t(z) \nabla _{\theta } \log \eta (z|s_t) + u_t(z) \nabla _{\theta } \log \pi _h(a_t|s_t) \right ) + \sum _{t=0}^{T-2} \left ( (u_t(z) - w_t(z)) \nabla _{\theta } \log \psi _h(s_{t+1}) + w_t(z) \nabla _{\theta } \log (1 - \psi _h(s_{t+1})) \right ) \right ).\]

        Following the Forward-backward algorithm, we can get the posterior for the latent i.e. \(\mathbb {P}((z_{t}, b_{t}) \mid \mathcal {H})\). We have the corresponding forward and backward components

        \[\begin {align*} \phi _t(z, b) &= \mathbb {P}(s_{0}, a_{0}, \ldots , s_{t}, z_{t} = z, b_{t} = b) \\ \beta _t(z, b) &= \mathbb {P}(s_{t + 1}, a_{t + 1}, \ldots , s_{T} \mid z_{t} = z, b_{t} = b), .\end {align*}\]

        which can then be written as

        \[\begin {align*} \phi _t(z, b) &= \mathbb {P}(s_t, a_t \mid z, b) \sum _{(z', b')} \phi _{t - 1}(z', b') \mathbb {P}(z, b \mid z', b') \\ \beta _t(z, b) &= \sum _{(z', b')} \beta _{t + 1}(z', b') \mathbb {P}(s_{t + 1}, a_{t + 1} \mid z', b') \mathbb {P}(z', b' \mid z, b) .\end {align*}\]

        Forward pass. For the forward pass, we have the initial condition

        \[\phi _0(z, b) = p_{0}(s_{0}) \delta _{\{b = 1\}} \eta (z \mid s_{0})\]

        Here \((z', b')\) ranges over the possible values of \((z_{t - 1}, b_{t - 1})\). It is sufficient to expand the remaining two terms in terms of our generative model as defined above. First, note that \((z_{t}, b_{t})\) are independent of \(b_{t - 1}\). Therefore, we can drop the \(b_{t - 1}\) term and are left with the following two cases:

        \[\begin {align*} \mathbb {P}_{\theta }(z, b = 0 \mid z') &= \delta _{\{z = z'\}} (1 - \psi _{\theta }(s_{t}, z')) \\ \mathbb {P}_{\theta }(z, b = 1 \mid z') &= \eta _{\theta }(z \mid s_{t}) \psi _{\theta }(s_{t}, z') .\end {align*}\]

        Plugging this in, we now have

        \[\begin {align*} \phi _t(z, b) = \mathbb {P}_{\theta }(s_t, a_t \mid z, b) \sum _{(z', b')} \phi _{t - 1}(z', b') \Big (&\eta _{\theta }(z \mid s_{t}) \psi _{\theta }(s_{t}, z') \\ &+ \delta _{\{z = z'\}} (1 - \psi _{\theta }(s_{t}, z'))\Big ) .\end {align*}\]

        It remains to find \(\mathbb {P}_{\theta }(s_t, a_t \mid z, b)\). Once again, given \(z_t\) we no longer care about \(b_t\). Therefore, this becomes

        \[\begin {align*} \mathbb {P}_{\theta }(s_t, a_t \mid z) &= \pi _{\theta }(a_t \mid s_t, z) p(s_t \mid s_{t - 1}, a_{t - 1}) .\end {align*}\]

        This gives us our final expression

        \[\begin {align*} \phi _t(z, b) = &\pi _{\theta }(a_t \mid s_t, z) p(s_t \mid s_{t - 1}, a_{t - 1}) \\ &\times \sum _{(z', b')} \phi _{t - 1}(z', b') \Big (\eta _{\theta }(z \mid s_{t}) \psi _{\theta }(s_{t}, z') + \delta _{\{z = z'\}} (1 - \psi _{\theta }(s_{t}, z'))\Big ) .\end {align*}\]
        Remark. We have abused notation here, allowing \(\mathbb {P}(s_t, a_t \mid z_t, b_t)\) to depend on \(s_{t - 1}, a_{t - 1}\), etc. This is to align with the notation from Forward-backward algorithm. It makes no difference in practice as long as the backward dependence is finite / bounded. However, in practice you would want the history dependence to be small and here it is only one-step.

        Backward pass. For the backward pass, we define the end term as

        \[\beta _{T-1}(z, b) = \pi (a_{T - 1} \mid z, s_{T - 1}) p(s_{T} \mid s_{T - 1}, a_{T - 1})\]

        Here \((z', b')\) ranges over the possible values of \((z_{t + 1}, b_{t + 1})\). The other terms are analagous i.e.

        \[\begin {align*} \mathbb {P}_{\theta }(z', b' = 0 \mid z) &= \delta _{\{z' = z\}} (1 - \psi _{\theta }(s_{t + 1}, z)) \\ \mathbb {P}_{\theta }(z', b' = 1 \mid z) &= \eta _{\theta }(z' \mid s_{t + 1}) \psi _{\theta }(s_{t + 1}, z) .\end {align*}\]

        and

        \[\begin {align*} \mathbb {P}_{\theta }(s_{t + 1}, a_{t + 1} \mid z') &= \pi _{\theta }(a_{t + 1} \mid s_{t + 1}, z') p(s_{t + 1} \mid s_t, a_t) .\end {align*}\]

        The setup for Learning Options via Compression.

        • A state abstraction posterior \(q_{\theta }(s_{t} \mid z_{t}, x_{t})\) and decoder \(p_{\theta }(a_{t} \mid s_{t})\), where \(\pi _{\theta }(a_t \mid s_t, x_t) = \mathbb {E}_{q_{\theta }(s_{t} \mid x_{t}, z_{t})}[p_{\theta }(a_t \mid s_t)]\)
          • This seems to be needed to ensure that the decoder cannot overfit to the offline data?
        • A termination network \(\beta _{\theta }(b_t \mid x_{1}, \ldots , x_{t})\)
        • A skill posterior \(\eta _{\theta }(z_{t} \mid z_{t - 1}, b_{t}, x_{1:T},a_{1:T})\).

        Unlike in DDO, where they do exact posterior inference, here we do variational inference. Namely, they directly try to model the posterior via the neural network \(\eta _{\theta }(z_{t} \mid z_{t - 1}, b_{t}, x_{1:T},a_{1:T})\). This has a few consequences:

        • The expectation gradient is now true Expectation Maximization. In particular, we need to optimize the ELBO.

        The ELBO of LOVE can be written as

        \[\begin {align*} \mathcal {L}_{\text {ELBO}}(\theta ) \approx \sum _{t=1}^{T} \log &p_{\theta }(\mathbf {a}_{t} \mid \mathbf {s}_{t}) + D_{\text {KL}}(q_{\theta }(m_{t} \mid \mathbf {x}_{1:t}) \parallel p_{\theta }(m_{t} \mid \mathbf {s}_{t})) \\ &+ D_{\text {KL}}(q_{\theta }(z_{t} \mid m_{t}, \mathbf {x}_{1:T}, \mathbf {a}_{1:T}) \parallel p(z_{t})) + D_{\text {KL}}(q_{\theta }(s_{t} \mid z_{t}, \mathbf {x}_{t}) \parallel p_{\theta }(s_{t} \mid s_{t-1}, m_{t-1})) .\end {align*}\]

        Notably this requires a few additional prior networks for the KL regularization term in the ELBO.

        • A boundary prior \(\beta (b_t \mid s_{t})\).
        • A skill prior \(p(z)\).
        • A state abstraction transition prior \(p(s_t | s_{t-1}, z_t)\).

        Additionally, we also need to sample the \(z\)'s. This is done via Gumbel softmax and causes much higher variance.

        Frankenstein-ing LOVE and DDO

        • July 23, 2026
        • Kellen Kanarios

        We find ourselves in-between LOVE and DDO. In particular, in DDO, they are concerned with learning a hierarchy of policies where the higher-level meta policy \(\eta \) must be usable online only given the current state.

        • (+) The learned termination conditions \(\beta \) are dependent on the active skill.
        • (+) Exact inference \(\implies \) lower variance.
        • (-) Weak skill model \(\eta (z \mid s)\).
        • (-) No explicit encouragement for temporally-extended skills.

        On the contrary, in LOVE, they treat skills as a pure sequence-modeling problem. The posterior \(q_{\theta }(z_{t} \mid z_{t - 1}, b_{t}, x_{0}, \ldots , x_{T})\) is purely used to label offline trajectories and give the skills meaning. To make them usable online, it is then required to learn an additional hierarchical policy over these skills.

        • (+) The skill-labeler is allowed to use not only the current state but also the future visited states.
        • (+) Compression objective encourages temporally extended skills.
        • (-) Design inherently does not allow the terminations to be skill-dependent.
        • (-) Need to sample \(z\)'s to compute gradients \(\implies \) higher-variance.

        Our graphical model. We try to combine the desirable properties.

        We essentially combine the insight of LOVE where instead of trying to model the full joint distribution i.e. \(\mathbb {P}(\mathcal {H}, \xi )\) we instead model \(\mathbb {P}(\mathcal {A}, \xi \mid \mathcal {S})\), where we treat the entire state history as known.

        In the language of HMMs, the emission probabilities only correspond to the actions. Therefore, we can perform the Forward-backward algorithm and condition on whatever component of the state we want without breaking the exactness.

        Remark. This is only a good idea for us because we do not need to use the meta controller during evaluation.

        Deliberation cost. In LOVE, they encourage temporal extension through the compression loss. This requires an empirical approximation of the skillmarginal \(p(z)\). For me, this caused the skill entropy to just collapse to zero instead of encouraging temporally extended options. Instead, we use a deliberation cost more common in the options literature The Option-Critic Architecture, where we just add the switch-rate as an additional loss term.

        • We are given the posterior \(\mathbb {P}(z_t = z, b_t = 1 \mid \mathcal {H})\). In our implementation this would be \((B, T, K)\)
        • We marginalize over skills \(z\) by summing to get \(\mathbb {P}(b_t = 1 \mid \mathcal {H})\)
        • Then average over batch \(B\) and time \(T\) to get expected reads.

        As in LOVE, we need to balance minimizing log-likelihood with compression / switching cost. To do so, we do dual descent.

        • First, without compression, we run the algorithm and observe what is approximately the best attainable log-likelihood loss is.
        • Then, we minimize the deliberation cost subject to \(\mathcal {L}_{\text {likelihood}} \leq C + \epsilon \)

Intuitively, this would end up with a higher-level policy that best utilizes the temporally-extended skills to reach the nearby region of \(g\) then concedes to the lower-level policy.

        The setup for Multi-Level Discovery of Deep Options:

        • We have a meta-controller that outputs the skill \(z\) given the state \(s\) i.e. \(z_t \sim \eta (\cdot \mid s_t)\)
        • We have the skill, which is a skill-conditioned policy \(a_t \sim \pi (\cdot \mid s_t, z_t)\)
        • We have the termination function \(\psi (\cdot \mid s_t, z_{t - 1})\), which is per skill and evaluates the previous skill \(z_{t - 1}\) at the new state \(s_t\), such that \(b_t \sim \mathrm {Ber}\big (\psi (s_t, z_{t - 1})\big ) \in \{0, 1\} \)
          • For \(b_t = 1\), we resample a new skill \(z_t \sim \eta (\cdot \mid s_t)\)
          • Otherwise, \(z_{t} = z_{t - 1}\)

        Assuming there is a true underlying latent history \(\xi = \{b_{0}, z_{0}, b_{1}, z_{1}, \ldots \} \) and the observable history \(\mathcal {H} = \{s_{0}, a_{0}, s_{1}, a_{1}, \ldots \} \), the likelihood according to the above generative model is given by

        \[\mathbb {P}(\mathcal {H}, \xi ) = p(s_{0})\delta _{\{b_{0} = 1\}}\eta (z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi (a_t \mid s_t, z_t) p(s_{t+1} \mid s_t, a_t)\]

        Furthermore, according to our model, we have that

        \[ \begin {align*} \mathbb {P}(b_t = 0, z_t \mid z_{t - 1}, s_t) &= (1 - \psi (s_t, z_{t - 1})) \cdot \delta _{\{z_{t} = z_{t - 1}\}} \\ \mathbb {P}(b_t = 1, z_t \mid z_{t - 1}, s_t) &= \psi (s_t, z_{t - 1}) \cdot \eta (z_t \mid s_t) .\end {align*} \]

        Up to this point, we have assumed this is the ground truth model and we are given each of the components. Instead we want to learn these components. To do so, we introduce our set of parameters \(\mathbf {\theta } \), such that each component is parametrized by a subset of \(\mathbf {\theta } \) i.e. \(\eta _{\theta }, \pi _{\theta }, \psi _{\theta }\).

        Goal. We want to learn some \(\theta \) that maximized the likelihood of the observable data.

        To do so, first consider the log-likelihood of our new parametrized model

        \[ \begin {align*} \log \mathbb {P}_{\theta }(\mathcal {H}, \xi ) &= \log \left [p(s_{0})\delta _{\{b_{0} = 1\}}\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t) p(s_{t+1} \mid s_t, a_t)\right ] \\ &= \log p(s_{0})\delta _{\{b_{0} = 1\}} + \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t) \right ] + \sum _{t=0}^{T-1} \log \left [p(s_{t+1} \mid s_t, a_t)\right ] \\ &= \underbrace {\log p(s_{0})\delta _{\{b_{0} = 1\}} + \sum _{t=0}^{T-1} \log \left [p(s_{t+1} \mid s_t, a_t)\right ]}_{\text {independent of } \theta } + \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t)\right ] .\end {align*} \]

        Since we are only concerned with learning \(\theta \), we can disregard the \(\theta \) independent terms leaving

        \[\begin {align*} \nabla _{\theta } \log \mathbb {P}_{\theta }(\mathcal {H}, \xi ) &= \nabla _{\theta } \left [ \log \left [\eta _{\theta }(z_{0} \mid s_{0}) \prod _{t=1}^{T - 1} \mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t) \prod _{t = 0}^{T - 1} \pi _{\theta }(a_t \mid s_t, z_t)\right ] \right ]\\ &= \nabla _{\theta } \log \left [\eta _{\theta }(z_{0} \mid s_{0})\right ] + \sum _{t=1}^{T - 1} \nabla _{\theta } \log \left [\mathbb {P}_{\theta }(b_t, z_t \mid z_{t - 1}, s_t)\right ] + \sum _{t = 0}^{T - 1} \nabla _{\theta } \log \left [\pi _{\theta }(a_t \mid s_t, z_t)\right ]. \end {align*}\]
        Problem. This requires samples \((\mathcal {H}, \xi ) \sim P_{\theta }(\mathcal {H}, \xi )\), but we only have samples \(\mathcal {H} \sim P_{\theta }(\mathcal {H})\)!!!
        Solution. We can use the trick from Expectation Maximization. Referred to in the paper as the expectation gradient trick. Namely, \[\nabla _{\theta } \log P_{\theta }(\mathcal {H}) = \mathbb {E}_{\xi \sim P_{\theta }(\xi \mid \mathcal {H})} \left [ \nabla _{\theta } \log P_{\theta }(\mathcal {H}, \xi ) \right ]\]

        Now it remains to compute the expectation from \(P_{\theta }(\xi \mid \mathcal {H})\)

        Definition. Hidden markov model [3C07]

        • July 21, 2026
        • Kellen Kanarios

        A hidden markov model is a statistical model, where the hidden states \(h \in \mathcal {H}\) form a Markov Chain, such that \(\mathbb {P}(h_{t+1} \mid h_{1}, \ldots , h_{t}) = \mathbb {P}(h_{t + 1} \mid h_{t})\)

        Unlike a Markov chain, in addition to the hidden states, a hidden markov models contains observations \(x \in \mathcal {X}\) with associated emission probabilities \(\mathbb {P}(x_t \mid h_t)\).

        From the graphical model, we can see that for any \(i < j\), \((x_{0}, \ldots , x_i) \perp (x_{j}, \ldots , x_{N}) \mid z_{i}\). Or in english, observations are independent conditioned on the hidden state.

        In our setup, we can rewrite the marginal as

        \[\mathbb {P}(z_t \mid \mathcal {H}) = \mathbb {P}(z_t \mid x_{0}, \ldots , x_{T})\]

        From Baye's rule, we can decompose this as

        \[\begin {align*} \mathbb {P}(z_t \mid x_{0}, \ldots , x_{T}) &= \frac {\mathbb {P}(x_{0}, \ldots , x_{T} \mid z_t)\mathbb {P}(z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})} \\ &= \frac {\mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t, x_{0}, \ldots , x_t) \mathbb {P}(x_{0}, \ldots , x_{t} \mid z_t)\mathbb {P}(z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})} \\ &= \frac {\mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t, x_{0}, \ldots , x_t) \mathbb {P}(x_{0}, \ldots , x_{t}, z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})} .\end {align*}\]

        Since we are considering an HMM, \((x_{1}, \ldots , x_t)\) and \((x_{t + 1}, \ldots , x_{T})\) are conditionally independent conditioned on \(z_t\). Thus, we can further simplify this to

        \[\mathbb {P}(z_t \mid x_{0}, \ldots , x_{T}) = \frac {\mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t) \mathbb {P}(x_{0}, \ldots , x_{t}, z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})}\]
        Notation. We will now consider the terms in the numerator separately. The first can be viewed as the backward term \[\beta _t(z) = \mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t = z),\] and the second as the forward term \[\phi _t(z) = \mathbb {P}(x_{0}, \ldots , x_{t}, z_t = z).\] Naming convention is to denote which direction in time information propagates (we will see shortly).

        Here \(z'\) ranges over the possible values of \(z_{t - 1}\). We see that \(\phi _t(z)\) depends only on information from time \(t\) and \(t - 1\). Therefore, information propagates forward in time.

         \[\begin {align*} \beta _t(z) &= \mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z) \\ &= \sum _{z'} \mathbb {P}(x_{t + 1}, \ldots , x_{T}, z' \mid z) \\ &= \sum _{z'} \mathbb {P}(x_{t + 2}, \ldots , x_{T} \mid x_{t + 1}, z', z) \mathbb {P}(x_{t + 1} \mid z', z) \mathbb {P}(z' \mid z) .\end {align*}\] First, since this is a HMM, everything is independent of \(z_{t}\) given \(z_{t + 1}\). Therefore, we can simplify \[\begin {align*} \beta _t(z) &= \sum _{z'} \mathbb {P}(x_{t + 2}, \ldots , x_{T} \mid x_{t + 1}, z') \mathbb {P}(x_{t + 1} \mid z') \mathbb {P}(z' \mid z). \end {align*}\] Additionally, \((x_{t + 2}, \ldots , x_{T}) \perp x_{t + 1} \mid z_{t + 1}\), such that \[\begin {align*} \beta _t(z) &= \sum _{z'} \mathbb {P}(x_{t + 2}, \ldots , x_{T} \mid z') \mathbb {P}(x_{t + 1} \mid z') \mathbb {P}(z' \mid z) \\ &= \sum _{z'} \beta _{t + 1}(z') \mathbb {P}(x_{t + 1} \mid z') \mathbb {P}(z' \mid z) .\end {align*}\]

        Here \(z'\) ranges over the possible values of \(z_{t + 1}\). We see that \(\beta _t(z)\) depends only on information from time \(t\) and \(t + 1\). Therefore, information propagates backward in time.

        Remark. Both passes are \(O(TK^2)\) because at each timestep, it requires computing \(\mathbb {P}(z_{t} \mid z_{t - 1})\) or \(\mathbb {P}(z_{t + 1} \mid z_{t})\) which is a \(K \times K\) transition matrix.

        Notably, the structure of the HMM ensures each marginal \(\mathbb {P}(z_t \mid \mathcal {H})\) depends only on the local transition probabilities \(\mathbb {P}(z_{t} \mid z_{t - 1})\) and \(\mathbb {P}(z_{t + 1} \mid z_{t})\) rather than the exponential global \(\mathbb {P}(z_{0}, \ldots , z_{T})\). This lends itself to the idea of a dynamic programming algorithm, where we compute the posterior of the preceding / succeeding latent and use the result for the next. Since we have bi-directional dependencies this requires making two passes a forward and backward pass. The key remaining detail is that these components can be efficiently combined to recover the full true marginal.

        Computing the posterior

        • July 21, 2026
        • Kellen Kanarios

        Finally, all that remains is to combine \(\phi \) and \(\beta \) to recover the true posterior. Recall that these directly correspond to the components in the numerator i.e.

        \[\mathbb {P}(z_t \mid \mathcal {H}) = \frac {\phi _t(z) \beta _t(z)}{\mathbb {P}(\mathcal {H})}\]

        Therefore, it remains to compute the normalization term \(\mathbb {P}(\mathcal {H})\). To do so, note that

        \[\begin {align*} \phi _t(z) \beta _t(z) = \mathbb {P}(\mathcal {H} \mid z) \mathbb {P}(z) \implies \sum _{z} \phi _t(z)\beta _t(z) &= \sum _{z} \mathbb {P}(\mathcal {H} \mid z) \mathbb {P}(z) \\ &= \mathbb {P}(\mathcal {H}) .\end {align*}\]

        Thus, the true posterior can be recovered as

        \[\mathbb {P}(z_t \mid \mathcal {H}) = \frac {\phi _t(z) \beta _t(z)}{\sum _{z} \phi _t(z)\beta _t(z)}\]

        Since \(z\) is discrete, this sum is only \(O(K)\) and therefore easily computable.

        Network routing strategies [VTS4]

        • May 12, 2026
        • Kellen Kanarios

        Given a topology and a packet's source and destination, which path through the network does it take? Three classes of answer:

        • Greedy (shortest-path) — always pick the shortest available route.
          • (+) Minimizes latency under light load.
          • (-) Overloads central links; worst-case throughput poor.
        • Uniform / oblivious random — pick a direction at random.
          • (+) Spreads load; avoids systematic hotspots.
          • (-) Wastes bandwidth on non-minimal paths; latency unpredictable.
        • Adaptive — at each hop, pick the outgoing link with the lowest local congestion.
          • (+) Dynamically balances load; best worst-case throughput.
          • (-) Routers need congestion state; risk of livelock without care.
        Example. Tornado traffic on an 8-node bidirectional ring: node \(i\) sends to \((i+3) \bmod 8\).
        • Greedy: all packets travel counter-clockwise. Bidirectional bandwidth wasted; \(\frac {1}{3}\) utilization.
        • Uniform: randomly clockwise/counter-clockwise. Counter-clockwise gets \(\frac {2}{3}\) utilization, clockwise \(\frac {2}{5}\); worst-case \(\frac {2}{5}\).
        • Adaptive: sends \(\frac {5}{8}\) counter-clockwise to match traffic exactly. Best.
        Remark. Adaptive routing tends to need Network backpressure signaling to know where congestion is, and care to avoid livelock when many adaptive decisions chase each other.
        Remark. Oblivious strategies like Dimension-order routing and Valiant's routing algorithm give deterministic guarantees (deadlock-free, bounded path length) at the cost of giving up the load-balancing of true adaptive routing.

        Definition. Torus topology [DDKT]

        • May 12, 2026
        • Kellen Kanarios

        A Mesh topology with wrap-around links connecting the ends of each dimension. In the \(k\)-ary \(n\)-cube classification, a \(k\)-ary \(n\)-cube. Uniform node degree (every node has degree \(2n\)), uniform load distribution, half the hop count of a mesh.

        • Every node has degree \(2n\).
        • Average hop count for even \(k\): \[H_{avg} = \frac {nk}{4}\]
        • Maximum channel load for uniform random traffic: \(k/8\) — half that of a Mesh topology.
        • Good path diversity: many shortest paths between any two nodes.
        Problem. Wrap-around links are physically long. To equalize wire delays, the torus must be folded, doubling effective wire lengths and link area. The wrap-around also doubles the link count versus a mesh of the same dimensions.
        Remark. Despite the theoretical advantage, mesh wins on real chips. Tori show up in HPC interconnects (Cray) where the wires can run on dedicated cables.

        Variants:

        • \(k\)-ary \(n\)-cube: \(k\) nodes along each of \(n\) dimensions, all ends wrapped.
        • Asymmetric: \((i, j, k)\)-ary — different node count per dimension.
        • Mesh: same as torus but without wrap-around.
        • Ring: \(k\)-ary 1-cube.

        Definition. Mesh topology [9SA2]

        • May 12, 2026
        • Kellen Kanarios

        A 2D grid of nodes with neighbors connected on the four compass directions, no wrap-around. The dominant on-chip topology in commercial designs (Xeon Phi, recent server x86 chips) because it lays out naturally on a 2D die.

        • Interior nodes degree 4; edge nodes degree 3; corners degree 2.
        • Average hop count for an even-\(k\) mesh: \[H_{avg} = \frac {nk}{3}\]
        • Maximum channel load for uniform random traffic: \(k/4\) — twice that of a Torus topology.
        Problem. Non-uniform node degree. Edge nodes carry less traffic than they could, while interior nodes bottleneck on traffic flowing through them.
        Remark. Mesh dominates Torus topology in practice despite the topology metrics being worse, because torus wrap-around wires are physically long and force expensive folding to equalize link lengths. Manufacturing wins over theoretical performance.

        Definition. Ring topology [UEA0]

        • May 12, 2026
        • Kellen Kanarios

        A direct network where each node connects to two neighbors, forming a closed loop. Bidirectional rings allow either direction of travel; unidirectional rings allow only one. The simplest topology to implement and reason about.

        In the \(k\)-ary \(n\)-cube family, a ring is a \(k\)-ary 1-cube.

        • Degree 2 (every node).
        • Diameter \(\lfloor k/2 \rfloor \) for a bidirectional \(k\)-node ring.
        • Bisection bandwidth 2 — cut the ring at exactly two links.
        • Hardware overhead very low: routers have two ports.
        Problem. Worst-case hop count is \(O(k)\). Doesn't scale beyond a small number of nodes.
        Remark. Despite the poor scaling, rings are common in commercial CPUs (Intel's L3 ring bus on Sandy/Ivy Bridge) because they're cheap and 8–16 cores is enough.

        Definition. Valiant's routing algorithm [XLDA]

        • May 12, 2026
        • Kellen Kanarios

        An oblivious Network routing strategies that achieves provably optimal worst-case throughput by introducing randomness in path selection:

        1. For a packet going from \(s\) to \(d\), randomly pick an intermediate node \(w\).
        2. Route \(s \to w\) using Dimension-order routing.
        3. Eject the packet at \(w\), then re-inject it.
        4. Route \(w \to d\) using Dimension-order routing.

        To preserve deadlock-freedom, the intermediate node must remove the packet from the network before re-injecting — otherwise the channel dependencies of the two halves could interlock.

        Remark. Restricting \(w\) to lie in the sub-quadrant defined by \(s\) and \(d\) keeps paths near-minimal while still spreading load — the practical compromise between guaranteed throughput and added latency.
        Remark. Valiant's algorithm trades average-case latency (paths are longer) for worst-case throughput (any traffic pattern looks roughly uniform once randomized). The right call for adversarial workloads; overkill for uniform traffic.

        Definition. Dimension-order routing [83P2]

        • May 12, 2026
        • Kellen Kanarios

        An oblivious Network routing strategies for \(k\)-ary \(n\)-cube networks (Mesh topology, Torus topology): always exhaust the offset in one dimension completely before changing to the next. E.g. in 2D mesh routing, route X-then-Y — never turn back to X after starting Y.

        The dimension order prevents routing deadlock: cycles in the channel dependency graph are impossible because no packet ever requests a channel in a "lower" dimension once it has started using a higher one. The dependency graph is therefore acyclic, and the standard cycle-equals-deadlock argument applies.

        • (+) Provably deadlock-free without virtual channels.
        • (-) Zero path diversity. Hot spots can't be routed around.
        • (-) Unaware of current network state — oblivious.

        The standard remedy for the path-diversity problem is Valiant's routing algorithm, which keeps the deadlock-freedom of dimension-order routing while spreading load with a randomized intermediate hop.

        Definition. Butterfly network [5WX9]

        • May 12, 2026
        • Kellen Kanarios

        A multistage indirect network. A \(k\)-ary \(n\)-fly has \(k^n\) terminal nodes connected through \(n\) stages of \(k \times k\) crossbar switches. At each stage, the routing decision selects which of \(k\) sub-groups to enter; after \(n\) stages, every terminal can reach every other.

        • Hop count \(n - 1\).
        • Under uniform random traffic, maximum channel load is 1 (perfectly balanced).
        • Channel load drops further for non-uniform traffic that respects the natural partitioning.
        Problem. Zero path diversity. There is exactly one path between any source and destination. A single hot link wedges traffic.
        Solution. Add shuffle stages that permute paths after each routing stage, so the same source-destination pair can be reached by multiple routes.

        Definition. Crossbar [YV5J]

        • May 12, 2026
        • Kellen Kanarios

        The simplest indirect network: an \(n \times m\) grid of switch points connecting \(n\) inputs to \(m\) outputs. Any input can be connected to any free output simultaneously — non-blocking.

        • One hop per transit (minimum possible).
        • Serializes if multiple senders target the same receiver.
        • Switch area grows as \(O(n \cdot m)\) — the dominant cost as systems grow.
        Remark. Crossbar router area scales as the square of the port count, which is why high-radix routers are dramatically more expensive than low-radix ones.

        Crossbars are common as the inner switch of larger structured networks (Clos network, Fat tree) — large flat crossbars are impractical at scale, but small crossbars are cheap.

        Definition. Fat tree [DFC7]

        • May 12, 2026
        • Kellen Kanarios

        A tree topology with bandwidth that doubles at every level toward the root. Each interior switch has logical degree 4, but as you climb the tree, link width doubles to absorb the combined traffic of its subtree — preserving bisection bandwidth.

        Structurally, a fat tree is a folded Clos network: the input and output stages of a Clos are merged into a single bidirectional stage at each level of the tree, and the middle stage becomes the higher levels of the tree.

        • Each switch has logical degree 4 (2 down, 2 up — with link doubling).
        • Bisection bandwidth scales linearly with the number of leaves.
        • Path diversity grows toward the root.
        Remark. Fat trees are the dominant topology for off-chip HPC interconnects (InfiniBand fat-tree clusters). Less common on-chip because the doubling links are hard to lay out on a 2D die.

        Definition. Clos network [YKAN]

        • May 12, 2026
        • Kellen Kanarios

        A three-stage indirect network parameterized by \((m, n, r)\): \(r\) input switches and \(r\) output switches each of size \(n \times m\), with \(m\) middle-stage switches each \(r \times r\). Total terminal nodes \(N = nr\). Non-blocking when \(m \geq 2n - 1\).

        • Number of total switching elements scales as \(N^{1.5}\) versus \(N^2\) for a flat Crossbar while preserving non-blocking.
        • Path diversity is \(m\) — multiple distinct paths between any source and destination.
        Proof. There are \(rmn\) input-switch elements, \(mr^2\) middle-switch elements, and \(rmn\) output-switch elements: \[S = 2rmn + mr^2.\] With \(r = N/n\): \[S = 2Nm + m(N/n)^2.\] Choosing \(n = \sqrt {N}\) and \(m \geq 2n - 1 = O(\sqrt {N})\): \[S = 4mN = O(N^{1.5}).\]
        Remark. A folded Clos network — input and output stages combined, making \(n\) input ports and \(m\) output ports bidirectional — is structurally the same as a Fat tree.

          In our setup, we can rewrite the marginal as

          \[\mathbb {P}(z_t \mid \mathcal {H}) = \mathbb {P}(z_t \mid x_{0}, \ldots , x_{T})\]

          From Baye's rule, we can decompose this as

          \[\begin {align*} \mathbb {P}(z_t \mid x_{0}, \ldots , x_{T}) &= \frac {\mathbb {P}(x_{0}, \ldots , x_{T} \mid z_t)\mathbb {P}(z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})} \\ &= \frac {\mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t, x_{0}, \ldots , x_t) \mathbb {P}(x_{0}, \ldots , x_{t} \mid z_t)\mathbb {P}(z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})} \\ &= \frac {\mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t, x_{0}, \ldots , x_t) \mathbb {P}(x_{0}, \ldots , x_{t}, z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})} .\end {align*}\]

          Since we are considering an HMM, \((x_{1}, \ldots , x_t)\) and \((x_{t + 1}, \ldots , x_{T})\) are conditionally independent conditioned on \(z_t\). Thus, we can further simplify this to

          \[\mathbb {P}(z_t \mid x_{0}, \ldots , x_{T}) = \frac {\mathbb {P}(x_{t + 1}, \ldots , x_{T} \mid z_t) \mathbb {P}(x_{0}, \ldots , x_{t}, z_t)}{\mathbb {P}(x_{0}, \ldots , x_{T})}\]
          Notation. We will now consider the terms in the numerator separately.
