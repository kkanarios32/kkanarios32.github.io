    Contrastive Learning [0009]

    Recall the longstanding objective of machine learning i.e. given data \(\mathcal {X}\) find the distribution \(p(x)\) that generated the data \(\mathcal {X}\). The most fundamental approach is maximum likelihood estimation. Namely, we want to maximize the log-likelhood of the data \[\max _{\theta } \sum _{x \in \mathcal {X}} \log p_{\theta }(x) \]

    NCE

    • October 31, 2024

    Suppose we are given \(N\) dog samples \(\mathcal {D}\) from some distribution \(p_d(x)\). We want to learn \(\theta \) so that \(p_{\theta }(x) \approx p_d(x)\). The idea is to learn to distinguish the dogs from our dataset and random samples according to some noisy distribution \(g(x)\). We can now return to the supervised learning setting, where we treat \(\mathcal {D}\) and \(\mathcal {R}\) as two classes. If we recall standard supervised learning practice, given a sample \(x\), we can easily estimate \[p(x \text { is a dog}) = 1 - p(x \text { is random noise})\] via binary classification.

    Explicitly, we will use logistic regression with the parametrization \[p_{\theta }(x) = \frac {1}{1 + e^{-G_{\theta }(x)}}.\] However, solving the logistic regression problem will yield \(p_{\theta ^*}(x) \approx p(x \text { is a dog})\). Recall that we want to estimate \(p_d(x)\) which is the distribution over the "dog manifold".

    To estimate the correct quantity, we need to leverage our knowledge of the noise distribution. Recall that if \(p_{\theta }(x) = p(\mathcal {D} \mid x)\) then \(G_{\theta }(x) = \log \frac {p(x \mid \mathcal {D})}{p(x \mid \mathcal {R})}\). Since we generated the samples from \(\mathcal {R}\), we have the explicit distribution i.e. \(p(x \mid \mathcal {R}) = p_{\mathcal {R}}(x)\). Therefore, we can restrict \(G_{\theta }\) to explicitly learn \(p(x \mid \mathcal {D})\) by considering \[G_{\theta }(x) = \log p_{\theta }(x \mid \mathcal {D}) - \log p_{\mathcal {R}}(x),\] considering the cross entropy loss we get the NCE loss

    Definition. NCE loss [000E]

    • November 4, 2024
    • Kellen Kanarios

    The NCE loss aims to minimize the following objective \[\mathcal {L}_{N} = - \sum _{t} \log \left [h(x_t; \theta )\right ] + \log \left [1 - h(y_t; \theta )\right ],\] where \(x_t\) are samples from the data distribution and \(y_t\) are randomly generated samples and \[\begin {array}{r c l}{{h({\bf u};\theta )}}&{{=}}&{{\frac {1}{1+\exp \left [-G({\bf u};\theta )\right ]},}}\\ {{G({\bf u};\theta )}}&{{=}}&{{\ln p_{m}({\bf u};\theta )-\ln p_{n}({\bf u}).}}\end {array}\]

    In the NCE paper, they show under mild conditions that the estimator \(p_{\theta }(x \mid D) \to p_{\mathcal {D}}(x)\) in probability as the number of samples in the loss goes to infinity. Equivalently, the estimator is consistent.

    Temporally Contrastive Learning in Time Series

    • October 31, 2024

    Before we get to contrastive RL, it is a natural question to wonder how does this apply to temporal sequences? Concretely, we want to make predictions about the future given the current "context". However, we want to do so in an unsupervised way, meaning we are only given trajectories not a notion of what it means for a trajectory to be good. Naively, one can try to do this in a supervised manner.

    For a \(k\) step prediction, this would just be your model predicting what will happen in \(k\) steps then seeing if it matches what occured \(k\) steps in the future in the sample trajectory. However, if your sample space \(\mathcal {X}\) is very high-dimensional, modeling this relationship can require an exorbinant amount of trajectories.

    This is a central question in the InfoNCE paper, where they propose the following loss:

    Definition. InfoNCE [000B]

    • November 4, 2024
    • Kellen Kanarios

    InfoNCE recasts representation learning as classification: given a context \(c_t\) and a batch \(\mathcal {X}\) holding one positive future sample \(x_{t+k}\) among \(N-1\) distractors, it trains a score \(f_k\) to pick the positive out of the batch. This is the workhorse objective of contrastive learning.

    The loss is the cross-entropy of that identification: \[\mathcal {L}_{N} = - \mathbb {E}_{\mathcal {X}} \left [\log \frac {f_k(x_{t + k}, c_t)}{\sum _{x_j \in \mathcal {X}} f_k(x_j, c_t)}\right ]\] The numerator rewards a high score on the true pair; the denominator normalizes over the positive plus every negative, so a score only counts by being large relative to the distractors. Minimizing \(\mathcal {L}_N\) maximizes a lower bound on the mutual information \(I(x_{t+k}; c_t)\), and the bound tightens with \(N\) — which is the mechanistic reason contrastive methods reward large batches and many negatives.

    Now we need to unpack this very ominous loss. To start, what are \(x_k\) and \(c_t\)?
