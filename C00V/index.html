Definition. Softmax [C00V]

  • July 14, 2025

The softmax function \(\mathrm {softmax}: \mathbb {R}^d \to \mathbb {R}^d\) is defined by \[\mathrm {softmax}(\mathbf {x})_i = \frac {e^{x_i}}{\sum _{j = 1}^{d} e^{x_j}}\]

Claim. The Jacobian of the softmax function for \(\mathbf {y} = \mathrm {softmax}(\mathbf {x})\) is given by \[ \begin {align*} \frac {\partial y_i}{\partial x_j} = y_i(\delta _{ij} - y_j), \end {align*} \] or, in matrix form, \[ \begin {align*} J = \mathrm {diag}(\mathbf {y}) - \mathbf {y}\mathbf {y}^\top . \end {align*} \]
Proof. Write \(y_i = e^{x_i} / S\) with \(S = \sum _{k=1}^{d} e^{x_k}\), so that \(\partial S / \partial x_j = e^{x_j}\). By the quotient rule, \[ \begin {align*} \frac {\partial y_i}{\partial x_j} &= \frac {\delta _{ij} e^{x_i} S - e^{x_i} e^{x_j}}{S^2} \\ &= \delta _{ij}\frac {e^{x_i}}{S} - \frac {e^{x_i}}{S}\frac {e^{x_j}}{S} \\ &= \delta _{ij} y_i - y_i y_j \\ &= y_i(\delta _{ij} - y_j). \end {align*} \] Collecting these entries into a matrix, the diagonal contributes \(y_i\) along \(\mathrm {diag}(\mathbf {y})\) and the \(-y_i y_j\) term is the outer product \(\mathbf {y}\mathbf {y}^\top \), giving \(J = \mathrm {diag}(\mathbf {y}) - \mathbf {y}\mathbf {y}^\top \).

    Flash Attention (Bwd) [OGCRZ]

    • May 27, 2026
    • Kellen Kanarios

    To understand the innovation of the backward pass in flash attention, we must first find all of the gradients we must compute.

    Notation. Following the notation in the original paper, we have the following matrices \[\mathbf {S} = \mathbf {Q} \mathbf {K}^T, \quad \mathbf {P} = \mathrm {softmax}(\mathbf {S}), \quad \mathbf {O} = \mathbf {P} \mathbf {V}\] Now suppose their is some scalar loss function \(\phi \). Then we define \[\frac {\partial \phi }{\partial \mathbf {O}}, \frac {\partial \phi }{\partial \mathbf {K}}, \frac {\partial \phi }{\partial \mathbf {V}} = d\mathbf {O}, d\mathbf {K}, d\mathbf {V}\]

    We will compute each gradient individually.

    1. \(d\mathbf {P}\): this is just a straightforward application of the chain rule. In particular, \(\mathbf {O} = \mathbf {P} \mathbf {V}\) so that \[ d\mathbf {P} = \frac {\partial \phi (\mathbf {O})}{\partial \mathbf {P}} = \frac {\partial \phi (\mathbf {O})}{\partial \mathbf {O}} \frac {\partial \mathbf {O}}{\partial \mathbf {P}} = d\mathbf {O} \mathbf {V}^T \]

      To sanity check, let us look at the shapes. \(d\mathbf {O}\) is the partial with respect to the outputs \(O\), so we need an entry for each output and it is \(N \times d\). \(V^T\) is \(d \times N\), so that the result \(d\mathbf {P}\) is \(N \times N\) as desired.

      Furthermore, we can see this by writing \(\mathbf {O}_{ik} = \sum _{j} \mathbf {P}_{ij} \mathbf {V}_{jk}\). Then

      \[ \begin {align*} d\mathbf {P}_{ij} &= \sum _{i} \frac {\partial \phi }{\partial \mathbf {O}_{ik}} \cdot \frac {\partial \mathbf {O}_{ik}}{\partial \mathbf {P}_{ij}} \\ &= \sum _{i} d\mathbf {O}_{ik} \mathbf {V}_{jk} .\end {align*} \]
    2. \(d\mathbf {V}\): this is just the same as \(d\mathbf {P}\). In particular, \(\mathbf {O} = \mathbf {P} \mathbf {V}\) so that \[ d\mathbf {V} = \frac {\partial \phi (\mathbf {O})}{\partial \mathbf {V}} = \frac {\partial \phi (\mathbf {O})}{\partial \mathbf {O}} \frac {\partial \mathbf {O}}{\partial \mathbf {V}} = \mathbf {P}^T d\mathbf {O} \]

      Furthermore, we can see this by writing \(\mathbf {O}_{ik} = \sum _{j} \mathbf {P}_{ij} \mathbf {V}_{jk}\). Then

      \[ \begin {align*} d\mathbf {V}_{jk} &= \sum _{i} \frac {\partial \phi }{\partial \mathbf {O}_{ik}} \cdot \frac {\partial \mathbf {O}_{ik}}{\partial \mathbf {V}_{jk}} \\ &= \sum _{i} d\mathbf {O}_{ik} \mathbf {P}_{ij} \\ &= \sum _{i} \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} d\mathbf {O}_{ik} .\end {align*} \]
    3. \(d\mathbf {S}\): now we apply chain rule with \(d\mathbf {P}\). However, we must take the gradient through the softmax. Exiting matrix notation hell, if we consider the scalar case. Let \(\mathbf {s}\) and \(\mathbf {p}\) just be rows. Then using the scalar softmax derivation \[ \begin {align*} d\mathbf {s}_j &= \sum _{i} d\mathbf {p}_i \cdot \frac {\partial \mathbf {p}_i}{\partial \mathbf {s}_j} \\ &= \sum _{i} d\mathbf {p}_i \cdot (\delta _{ij}\mathbf {p}_i - \mathbf {p}_{ij}) \\ &= d \mathbf {p}_{j} \mathbf {p}_{j} - \mathbf {p}_{j}\sum _{i} d\mathbf {p}_{i} \mathbf {p}_{i} .\end {align*} \] Now if we rewrite this as the row vector, we get \[ \begin {align*} d\mathbf {S}_{k} &= \mathbf {P}_{k} \odot (d\mathbf {P}_{k} - \mathbf {P}_k \cdot d\mathbf {P}_k) .\end {align*} \] To get back to the full matrix, define \(\mathbf {D}_i = \mathbf {P}_i \cdot d\mathbf {P}_i\) i.e. a vector whose \(i\)th entry is \(\mathbf {P}_i \cdot d\mathbf {P}_i\). Then we have that \[ \begin {align*} d\mathbf {S} &= \mathbf {P} \odot (d\mathbf {P} - \mathbf {D}) .\end {align*} \]
    4. \(d\mathbf {Q}\): To get \(d\mathbf {Q}\), we just apply the chain rule to \(d\mathbf {S}\). Since \(\mathbf {S} = \mathbf {Q} \mathbf {K}^T\), \[ \begin {align*} d\mathbf {Q}_{i} &= \sum _{j} d\mathbf {S}_{ij} \mathbf {K}_{j} \\ &= \sum _{j} \mathbf {P}_{ij} \cdot (d\mathbf {P}_{ij} - \mathbf {D}_i) \mathbf {K}_{j} .\end {align*} \]
    5. \(d\mathbf {K}\): Same as \(d\mathbf {Q}\) i.e. \[ \begin {align*} d\mathbf {K}_{j} &= \sum _{j} d\mathbf {S}_{ij} \mathbf {Q}_{i} \\ &= \sum _{j} \mathbf {P}_{ij} \cdot (d\mathbf {P}_{ij} - \mathbf {D}_i) \mathbf {Q}_{i} .\end {align*} \]
    Problem. \(\mathbf {P}\) and \(d\mathbf {P}\) are \(N \times N\) matrices. Materializing this is equivalent to materialing the attention matrix in the forward pass.
    Insight. recomputing \(\mathbf {P}_{ij}\) and \(d\mathbf {P}_{ij}\).

    First, we can easily recompute \(\mathbf {P}_{ij}\) with \(\mathbf {Q}\) and \(\mathbf {K}\) (which are both \(N \times d\) and must be resident in memory) via

    \[\mathbf {P}_{ij} = \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}}\]

    We can apply the same trick to \(d\mathbf {P}_{ij}\) with \(d\mathbf {O}\) and \(\mathbf {V}\) (both \(N \times d\)). Namely,

    \[d\mathbf {P}_{ij} = \sum _{k} d\mathbf {O}_{ik} \mathbf {V}_{jk} = d\mathbf {O}_{i}^T \mathbf {V}_j\]

    Never need an \(N \times N\) matrix.

    Now we can rewrite \(d\mathbf {Q}\) and \(d\mathbf {K}\) as \[ d\mathbf {Q}_{i} = \sum _{j} \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} \cdot (d\mathbf {O}_{i}^T \mathbf {V}_j - \mathbf {D}_i) \mathbf {K}_{j} \] and \[ d\mathbf {K}_{j} = \sum _{j} \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} \cdot (d\mathbf {O}_{i}^T \mathbf {V}_j - \mathbf {D}_i) \mathbf {Q}_{i} \]

    Problem. \(\mathbf {D}\) requires computing the global dot product \(\mathbf {P}_i \cdot d\mathbf {P}_i\).
    Insight.

    To get around this, we precompute \(\mathbf {D}\) using the \(\mathbf {O}\) values from the forward pass and the current \(d\mathbf {O}\) values. In particular,

    \[ \begin {align*} \mathbf {D}_i &= \mathbf {P}_{i} \cdot d\mathbf {P}_{i} \\ &= \sum _{j} \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} d\mathbf {P}_{ij} \\ &= \sum _{j} \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} d\mathbf {O}_{i}^{T} \mathbf {V}_{j} \\ &= \sum _{j} d\mathbf {O}_{i}^{T} \left (\frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} \cdot \mathbf {V}_{j}\right ) \\ &= d\mathbf {O}_{i}^{T} \sum _{j} \frac {\exp \left (\mathbf {Q}_{i} \cdot \mathbf {K}_{j}\right )}{\mathbf {L}_{i}} \cdot \mathbf {V}_{j} \\ &= d\mathbf {O}_{i}^{T} \mathbf {O}_{i} .\end {align*} \] Thus, we can compute \(\mathbf {D}_i\) directly from the vectors \(d\mathbf {O}_i\), \(\mathbf {O}_i\).

    Summary

    • May 27, 2026
    • Kellen Kanarios
    • In the forward pass,
      1. Compute and store \(\mathbf {O}\) values in memory.
      2. Compute and store \(\mathbf {L}\) values in memory
    • Between the forward and backward pass,
      1. Compute \(d\mathbf {O}\). (This is done automatically in JAX)
      2. Use \(\mathbf {O}\) and \(d\mathbf {O}\) to compute \(\mathbf {D}\). \(O(n)\) memory.
    • In the backward pass,
      1. Compute \(d\mathbf {V}_j\) via \(\mathbf {Q}, \mathbf {K}, d\mathbf {O}\). \(O(d)\) memory.
      2. Compute \(d\mathbf {Q}_i\) via \(\mathbf {Q}_i, \mathbf {K}_j, \mathbf {L}_i, \mathbf {D}_i\). \(O(d)\) memory.
      3. Compute \(d\mathbf {K}_j\) via \(\mathbf {Q}_i, \mathbf {K}_j, \mathbf {L}_i, \mathbf {D}_i\). \(O(d)\) memory.

    Online Softmax [PI7N]

    • May 20, 2026
    • Kellen Kanarios

    This covers the math behind the online softmax algorithm required by Flash Attention (v1).

    Safe softmax

    • May 20, 2026
    • Kellen Kanarios

    The first thing we must consider is numerical precision. Namely, the original softmax operation is given by

    \[\mathrm {softmax}(\mathbf {x}) = \frac {\exp (\mathbf {x})}{\sum _{i=1}^n \exp (\mathbf {x}_i)}\]

    However, when operating with low precision data types (i.e. float16) the maximum value is \(65536\). Therefore, with \(\exp \) we can easily get numerically overflow.

    To get around this, the online softmax algorithm adheres to the safe softmax. For safe softmax, we simply multiply the numerator and denominator by \(\exp (-m)\), where \(m = \max \{x_{1}, \ldots , x_n\}\). This gives

    \[\mathrm {softmax}(\mathbf {x}) = \frac {\exp (\mathbf {x} - m)}{\sum _{i=1}^n \exp (\mathbf {x}_i - m)}\]

    Since \(x_i - m \leq 0\), this ensures that each \(\exp (x_i - m)\) is bounded in a safe range.

    Problem. Note that computing the \(\max \) over \(\mathbf {x}\) means that we must have all of the logits in a row of \(QK^T\) materialized in SMEM, or first loop over all of them to compute the max before computing the normalization constant.

    Fusing the max and normalization

    • May 20, 2026
    • Kellen Kanarios

    The online softmax algorithm fuses the max reduction into the normalization pass, so a single sweep over the row suffices to compute the normalization with the max. The key difficulty is that the denominator depends on \(n\) for each \(i\) i.e.

    \[d = \sum _{i=1}^{n} \exp (x_i - m_n)\]

    Therefore, we must find a recursive relationship such that \(d_{i+1} = f(d_i, m_i, x_{i+1})\) for some function \(f\). This would then allow us to locally compute \(d_{i+1}\) with only the local information of current \(x_{i+1}\) and the two running statistics \(d_i\) and \(m_i\). This is exactly the kind of algorithm that is conducive to memory tiling.

    Question. What is \(f(d_i, m_i, x_{i+1})\)?
    Answer. Consider \(d_{i+1} = d_i \cdot \exp (m_{i} - m_{i + 1}) + \exp (x_{i + 1} - m_{i + 1})\).
    Proof. This can be seen by directly rewriting \(d_{i + 1}\). \[ \begin {align*} d_{i + 1} &= \sum _{j = 1}^{i+1} \exp (x_j - m_{i + 1}) \\ &= \left (\sum _{j = 1}^{i} \exp (x_j - m_{i+1})\right ) + \exp (x_{i+1} - m_{i+1}) \\ &= \left (\sum _{j = 1}^{i} \exp (x_j - m_{i})\right )\exp (m_{i} - m_{i + 1}) + \exp (x_{i+1} - m_{i+1}) \\ &= d_{i} \exp (m_{i} - m_{i + 1}) + \exp (x_{i+1} - m_{i+1}) \end {align*} \]

    The running denominator \(d\) is rescaled by \(e^{m - m_{\text {new}}}\) each time a new maximum is found, keeping it consistent with the safe-softmax shift.

    Remark. Unfortunately, we cannot get around the second pass to compute the normalized output for \(\mathbf {p}\). However, this is not the case in Flash Attention (v1).

    Tokenization [0087]

    • July 1, 2025
    • Kellen Kanarios

    Introduction

    • July 1, 2025
    • Kellen Kanarios

    A tokenizer is responsible for taking text and turning it into a numerical representation we can pass to a neural network i.e. \[\mathsf {encode}(\text {hello world}) \mapsto \begin {bmatrix} 24912, & 2375 \end {bmatrix} \] However, we also need \[\mathsf {decode}(\begin {bmatrix} 24912, & 2375 \end {bmatrix}) \mapsto \text {hello world} \] When defining a "good" tokenizer, we are concerned with the compression ratio i.e. the # of bytes per token.

    Character-based tokenization

    • July 1, 2025
    • Kellen Kanarios

    Trivially, we can do character-based tokenization i.e. mapping each character to a unique number. For example, \(\mathsf {chr}(97) = \text {``a''}\) and \(\mathsf {ord}(\text {``a''}) = 97\). The compression ratio will be \(> 1\) because each token corresponds to a character potentially consisting of more than one byte.

    Problem.
    1. Very large vocabulary
    2. Many characters are quite rare.
    Remark. There is a fundamental tradeoff between compression ratio and vocabulary size. As an example, suppose we map every sequence of \(n\) bytes to a token. This would achieve a compression ratio of \(n\). However, this would require \(256^n\) tokens in our vocabulary.

    Problem 2 was not entirely straight-forward to me. This actually reminds me of an idea from my information theory course, where we can assume we are given a "prior" over human language then we want to map low-probability characters to large strings in order to minimize the expected length (see huffman coding).

    Byte-based tokenization

    • July 1, 2025
    • Kellen Kanarios

    In byte-based tokenization, each byte is a token achieving a compression ratio of exactly \(1\), where a byte \(\mapsto [0, 256]\) . This is very elegant in the sense that our vocabulary size is fixed at \(256\).

    Problem. However, with this, the length of our encoding is the number of bytes in the string, which will become too long.

    Word-based tokenization

    • July 1, 2025
    • Kellen Kanarios

    In word-based tokenization, we simply split the string into the corresponding words then map each of these to a unique integer.

    Problem. The vocabulary size is "unbounded" i.e. if we see a word we have not seen before then we need to extend the vocabulary size.

    Byte Pair Encoding

    • July 1, 2025
    • Kellen Kanarios

    For the byte pair encoding (BPE), we instead train the tokenizer on raw text to automatically determine the vocabulary.

    Remark. In GPT2, they first split the raw text into words and then do this byte pair encoding.
    Algorithm. while true:
    1. Count occurences of byte pairs. Store in dict \(D\) with \(\{(b_1, b_2): \text {count}\}\)
    2. Find \((b_1, b_2)\) with highest count.
    3. Merge \((b_1, b_2)\) by \((b_{1}, b_{2}) \mapsto b'\)
    4. Replace all instances of \((b_{1}, b_{2})\) in string with \(b'\) i.e. \[b_{1} b_{2} b_{3} b_{4} \mapsto b'b_{3} b_{4}\]
    Remark. There are a few additional implementation details to consider for project 1.
    1. \(\mathsf {encode}\) loops over all merges. Only loop over merges that matter.
    2. Detect and preserve special tokens (e.g. <|endoftext|>)
    3. Use pre-tokenization (GPT2 regex)

    Motivating Questions

    • July 1, 2025
    • Kellen Kanarios

    We start by discussing a few essential questions one hopes to answer when they set out to train a large model.

    Question. How long to train a 70b parameter model on 15T tokens on 1024 H100s?
    Answer. Mysteriously, we spawn from the ether (will be explained later) that total_flops = 6 * 70e9 * 15e12. Intuitively, we know that we need to perform some computation for each parameter for each token. However, the 6 is still an ominous mystery. Given the total flops, we then can just take the GPUs FLOP/s multiply by the number of seconds in the day and divide total_flops by this number i.e. days = total_flops / flops_per_day
    Remark. In practice, we must also consider the model flop utilization (MFU). Basically, our algorithm cannot utilize the entirety of the maximum FLOP/s provided by the hardware due to things like memory bandwidth, etc. To account for this in the calculation, they just take flops_per_day / 2.
    Question. What is the largest model that you can train on 8 H100s using AdamW?
    Answer. An H100 has 80gb of shared memory i.e. h100_bytes = 80e9. The calculation then proceeds as 
    # parameters, gradients, optimizer state
bytest_per_param = 4 + 4 + (4 + 4)
num_parameters = (h100_bytes * 8) / bytes_per_parameter
    what we need each of parameters, gradients, optimizer state will become clear later.
    Remark. In practice, we also need to account for activations, which will depend on both batch size and sequence length.

    Memory Accounting [DEQU]

    • July 3, 2025

    Tensor Memory: Tensors are stored as a floating point number. Memory is determined by (i) number of values in tensor and (ii) data type of each value.

    Low Precision Data Types [DEQV]

    • July 3, 2025

    float16: here, we decrease both the fraction and exponent bits. Importantly, there are only \(5\) exponent bits making the effective range \((2^{-6}, 2^{6})\). The dynamic range for small numbers is pretty bad. This can cause small numbers to "die" effectively killing these neurons for the remainder of learning due to backpropagation.

    bfloat16: to fix the problems with float16, they instead keep \(8\) exponent bits and reduce the fraction bits to \(7\). This maintains the same dynamic range as float32 while still only using \(16\) bits.

    fp8: Two variants

    1. E4M3: has 4 exponent bits and 3 mantissa bits
    2. E5M2: has 5 exponent bits and 2 mantissa bits
    Requires modern Hopper Architecture i.e. H100 to perform operations. Also supported by TPUs.

    Training Implications

    1. float32 training works but requires lots of memory
    2. Using lower precision data types allows you to train larger models due to lower memory requirements, but can cause instability
    3. To optimize this tradeoff, use mixed-precision training.

    Torch Tensors [DYIV]

    • July 3, 2025

    In pytorch, tensors are contiguous blocks of memory, where their shape is controlled by strides. A stride basically just refers to how far you have to jump in physical memory to correspond to incrementing the logical index i.e. in a (2, 3) matrix each row requires jumping \(3\) spots in physical memory. Thus, stride[1] = 3.

    Remark. Calling .view((m, n)) just changes the stride but does not modify the physical memory. If you call .view that results in a non-contiguous view, then you cannot call .view again because it assumes contiguous indexing to properly adjust the strides. To deal with this, they have .contiguous() and .reshape() is basically .contiguous().view().
    Remark. All elementwise operations make a copy i.e. x + y. A useful function for masked attention is triu().

    Definition. RMS Norm [1XNO]

    • July 14, 2025

    The RMS norm is simply the layer norm without the added bias term and centering i.e. \[\mathrm {RMSNorm}: \mathbb {R}^d \to \mathbb {R}^d, \quad \mathbf {x} \mapsto \frac {\mathbf {x}}{\sqrt {\mathrm {Var}[\mathbf {x}_i] + \epsilon }}*\gamma .\] This is used to spare memory movement of layernorm, where the bias term \(\beta \in \mathbb {R}^d\) and has been found to be empirically almost if not as good.

    One might wonder if matrix multiplies are the only thing that matters what can such a small change really accomplish.

    • Due to memory movement, despite being .17% of FLOPS, provided 25% speedup in runtime!!
    Remark. Most people apparently just drop the bias term and keep the centering i.e. subtracting the mean. This makes sense because computing the mean does not require loading any additional information back to memory.

    Activations

    • July 13, 2025
    • Kellen Kanarios

    Despite a long list of activations, the two focused on are ReLU and GELU.

    Definition. Rectified Linear Unit (ReLU) [BEA2]

    • July 14, 2025

    The rectified linear unit(ReLU) is defined as \[\mathrm {ReLU} : \mathbb {R}^d \to \mathbb {R}^d, \quad \mathbf {x} \mapsto \max (0, \mathbf {x}),\] where the \(\max \) is done elementwise i.e. \(\mathrm {ReLU}(\mathbf {x})_i = \max (0, x_i)\) This provides "nice" gradients, making it common when training neural networks.

    Definition. Gaussian Error Linear Unit (GELU) [5SPM]

    • July 14, 2025

    The Gaussian Error Linear Unit (GELU) is a slight modification to the ReLU to account for the non-differentiability at \(0\). Namely, \[\mathrm {GELU}(\mathbf {x}) : \mathbb {R}^d \to \mathbb {R}^d, \quad \mathbf {x} \mapsto \mathbf {x} \cdot \psi (\mathbf {x}),\] where \(\psi (\mathbf {x}) = \mathrm {CDF}(\mathcal {N}(\mathbf {x}, \mathbf {I}))\)

    Definition. Swish Activation [OZD8]

    • July 14, 2025

    The Swish activation is given by \[\mathrm {Swish}(\mathbf {x}) : \mathbb {R}^d \to \mathbb {R}^d, \quad \mathbf {x} \mapsto \mathbf {x} s(\mathbf {x}),\] where \(s : \mathbb {R}^d \to \mathbb {R}^d\) is the sigmoid function.

    Remark. The extra computation required by GELU or Swish is a non-factor because FLOPS are dominated by matrix multiplication and there is no increase in memory pressure.

    Definition. Gated Linear Unit (GLU) [K2N7]

    • July 14, 2025

    A gated linear unit is an activation function combined with an elementwise multiplication i.e.

    • \(\mathrm {ReGLU} = \mathrm {ReLU}(\mathbf {x}) \otimes \mathbf {V} \mathbf {x},\)
    • \(\mathrm {SwiGLU} = \mathrm {Swish}(\mathbf {x}) \otimes \mathbf {V} \mathbf {x},\)
    where \(\mathrm {ReLU}\) is the ReLU activation and \(\mathrm {Swish}\) is the Swish activation respectively.

    Remark. The matrix \(\mathbf {V}\) introduces additional learnable parameters. Therefore, an architecture that uses gating typically reduces their other parameters by a factor of \(\frac {2}{3}\).

    Serial vs. Parallel

    • July 13, 2025
    • Kellen Kanarios
     Another small trick is to parallelize computation. Traditionally, one computes the \(\mathrm {MHA}\) output and then passes this to the \(\mathrm {FFN}\) i.e. \[\mathbf {y} = \mathbf {x} + \mathrm {FFN}(\mathrm {LN}(\mathbf {x} + \mathrm {MHA}(\mathrm {LN}(\mathbf {x})))).\] However, this requires waiting for the \(\mathrm {MHA}\) to complete before performing the \(\mathrm {FFN}\) computation. It has been found that performing these in parallel does not cause any severe degradation in performance. Explicitly, instead they do \[\mathbf {y} = \mathbf {x} + \mathrm {FFN}(\mathrm {LN}(\mathbf {x})) + \mathrm {MHA}(\mathrm {LN}(\mathbf {x}))\]

    Rotary Position Embeddings (ROPE) [TOD6]

    • July 14, 2025

    Notation: For this section, our notion of an embedding is a function \(f\) that takes a token and a position i.e. \(f(x, i)\) is the embedding of token \(x\), occurring at position \(i\).

    Idea: the inner product of two embeddings should only need relative positioning i.e. for any two \(\mathbf {x}\), \(\mathbf {y}\) with positions \(i\) and \(j\) respectively, there should exist some \(g\), such that \[\langle f(\mathbf {x}, i), f(\mathbf {y}, j) \rangle = g(\mathbf {x}, \mathbf {y}, i - j)\] Namely, this can be written as a function of just the relative positioning between the embeddings.

    The simplest transformation that only preserves this relative information are rotations.

    However, rotations are only easily defined for \(\mathbb {R}^2\). To get around this, they simply partition their input 2d slices and apply the rotation via \[\mathrm {ROT} = \begin {bmatrix} \cos m \theta _{1} & - \sin m \theta _{1} & & & \\ \sin m \theta _{1} & \cos m \theta _{1} & & & \\ & & \cos m \theta _{2} & -\sin m \theta _{2} & \\ & & \sin m \theta _{2} & \cos m \theta _{2} & \\ & & & & \ddots \end {bmatrix} \]

    Remark. The \(\theta _i\)'s are fixed and chosen according to some schedule to capture different "frequencies".

    The embedding are used by appling them to the query and key matrices separately i.e. \[W_K = \mathrm {ROT}(W_K), \quad W_Q = \mathrm {ROT}(W_Q)\] These are then what is used in self-attention.

    Hyperparameter Rules (CS336) [9VZH]

    • July 14, 2025

    Rule 1: \(d_{\text {ff}} = d_{\text {model}}\)

    • \(d_{\text {model}}\) is the input dimension
    • \(d_{\text {ff}}\) is the hidden dimension

    Exception 1: in the case of GLUs, we recall that to account for extra parameters we scale \(\frac {2}{3}\). Therefore, \(d_{\text {ff}} = \frac {8}{3}d_{\text {model}}\).

    Exception 2: In T5, they use \(d_{\text {ff}} = 64d_{\text {model}}\). The logic for this was that we could maximize the MFU by increasing matrix size. Ended up going back to small \(d_{\text {model}}\).

    Rule 2: \(d_{\text {head}} = d_{\text {model}} / \text {num heads}\)

    • I believe the \(d_{\text {head}}\) is the output dim of each head.
    • This just says the total output dim for \(n\) heads is the same as if we did one head.
    • Means there is something important about splitting up the heads.

    Rule 3: Aspect ratio = \(\frac {d_{\text {model}}}{n_{\text {layer}}} \approx 100-200\)

    • Pipeline dependent: if network speed is fast than parallelizing is easier and shallow networks are better.
    • If poor network speed then pipeline parallel might be more viable and deeper networks would be more parallelizable.

    Rule 4: Vocab size

    • For single language models, vocab size is typically \(30-50k\)
    • For multi language models, vocab size is typically \(100-250k\)

    Stability Tricks

    • July 1, 2025
    • Kellen Kanarios

    The softmax is ill-behaved due to the exponentials. In the transformer, we have two softmaxes: one at the end and one in the attention.

    • For the first, note \[\log \sigma (\mathbf {x})_i = \log (x_i) - \log \underbrace {\sum _{j=1}^{d} e^{x_j}}_{D(\mathbf {x})}\] The only problem is the denominator term. The idea is to enforce the \(D(\mathbf {x}) = 1\) by regularizing via a penalty on \(\log D(\mathbf {x})\) ie. \[\mathcal {L}_{\text {aux}} = 10^{-4} \log ^2(D(\mathbf {x}))\] With the \(\nabla D(\mathbf {x})\) should be \(0\) and we are effectively only considering the non-exponential term.
    • For the attention, they primarily operate on the logits prior to the softmax. One way is via layer norm before the softmax. Another is via softcapping i.e. \[\mathrm {logits} = \mathrm {softcap} \cdot \tanh \left ( \frac {\mathrm {logits}}{\mathrm {softcap}} \right ) \]

    Multi Query Attention [W4YP]

    • July 14, 2025
    • Kellen Kanarios

    Notes on the paper Fast Transformer Decoding: One Write-Head is All You Need.

    For,

    • \(b\) batch dimension,
    • \(n\) sequence length,
    • \(d\) hidden dimension,
    • \(h\) number of heads.
    The total FLOPS for attention are roughly \(bnd^2\). This comes from
    1. \(3nd^2\) FLOPS to compute \(Q = XW_Q, K = XW_K, V = XW_V\).
    2. Need to do this \(b\) times for each input in batch.
    and the total memory accesses are roughly \(\underbrace {bnd}_{X} + \underbrace {bhn^2}_{\text {softmax}} + \underbrace {d^2}_{\text {projection}}\). This gives high arithmetic intensity \[O\left (\left (\frac {h}{d} + \frac {1}{bn}\right )^{-1}\right )\]

    However, if we do incremental inference then we must multiply our total number of memory accesses by \(n\) i.e. \(bn^2d + nd^2\). This gives arithmetic intensity \[O\left (\left (\frac {n}{d} + \frac {1}{b}\right )^{-1}\right ),\] which requires large batch and short sequence length.

    Remark. I believe we just ignore the softmax memory contribution in the inference case because it does not scale with \(n\) anymore as we are computing the logits just for the next token. Therefore, it becomes a \(\frac {d}{h}\) term, which we can safely ignore?

    The key is that the \(\frac {n}{d}\) term comes from the \(bn^2d\) term, where we are loading \(h\) \((b \times n \times d / h)\) \(K\) and \(V\) matrices \(n\) times. Thus, we can improve the \(\frac {n}{d}\) term by a factor of \(h\) by simply not using a different \(K\) and \(V\) matrix for each head. This is the entire idea behind MQA.

    Remark. Similar to the KV-cache, we still can use \(h\) \(Q\) matrices because we do not need to load \(n\) of them into memory because only the last one matters for inference.

    Group Query Attention [XUVV]

    • July 14, 2025

    Group query attention is the same idea as MQA but instead of using one \(K\) and \(V\) for every head, they use a group of them. Where obviously the number of groups needs to be less than the number of heads.

    Also inspired by this idea of reducing the dependence of \(K\) and \(V\) on the sequence length is sliding window attention.

    Sliding Window Attention [WM3M]

    • July 14, 2025

    The idea behind sliding window attention is to ensure the number of weight matrices needed for the keys \(K\) and values \(V\) does not scale with the sequence length. Intuitively, this means allowing each word to "attend" to only some fixed number of previous words rather than the whole sequence
