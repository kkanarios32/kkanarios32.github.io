      Opening a file in UNIX [JQ3C]

      • July 9, 2026
      • Kellen Kanarios

      For more details, see [bartlettProgrammingGround].

      Open a file with the open syscall. Must be called with the following:

      • eax contains system call number.
      • ebx contains pointer to string with name of file to open.
      • ecx contains the options for opening the file.
      • edx contains permissions used to open the file.

      Pointers to all command line arguments are stored on the stack.

      • Number of arguments is stored at \(8\)(esp),
      • Name of program at \(12\)(esp),
      • Arguments at \(16\)(esp) on.

      These are the argv array in C.

      fd (int) → fd table → struct file → struct inode
   per-process         "open file        the actual file
                       description"      (one per file)

      What happens during a function call [OGBJ]

      • July 8, 2026
      • Kellen Kanarios

      For this note, we will assume we are using the C calling convention. More details in [bartlettProgrammingGround].

      The first thing we do is push the arguments onto the stack (in reverse order). Then execute a call instruction. The call instruction does two things:

      1. It pushes the return address onto the stack. This is the address of the next instruction after the function call.
      2. It then modifies eip to point to the function to be called.

      Now we have jumped to the function to be called. However, there is still some bookkeeping that must be done within a function. Namely, we must keep track of where the stack started on a function call. To do so, we copy esp (stack pointer) into ebp (base register). This allows us to access each parameter stored to the stack by the caller via \(N*4 + 4\) ebp.

      Finally, we simply allocate space on the stack by subtracting the size of the arguments from esp. This is the new stack pointer.

      Now we have to return from the function. To do so, we first store the return value in the special register eax. Then, we must return to where we left off prior to calling the function. Recall that we saved the next instruction address at the top of the stack when we first called the function. There are now other local variables, etc. in the stack so we have to unwind it back. We can do this simply by 

      movl %ebp, %esp
popl %ebp
ret
      Here, the ret instruction just pops whatever is on the top of the stack and sets rip to it.

      We are now back in the calling code with the return value stored in eax. However, remember that the caller pushed the function parameters onto the stack. Therefore, we must also pop all of these parameters off before continuing execution.

      Components of a function [VSAV]

      • July 8, 2026
      • Kellen Kanarios

      Summarized from [bartlettProgrammingGround].

      Local variables. Local variables are data storage that a function uses while processing that is thrown away when it returns. It’s kind of like a scratch pad of paper. Functions get a new piece of paper every time they are activated, and they have to throw it away when they are finished processing. Local variables of a function are not accessible to any other function within a program.

      Static variables. Static variables are data storage that a function uses while processing that is not thrown away afterwards, but is reused for every time the function’s code is activated. This data is not accessible to any other part of the program. Static variables are generally not used unless absolutely necessary, as they can cause problems later on.

      Global variables. Global variables are data storage that a function uses for processing which are managed outside the function.       Return address. The return address is an "invisible" parameter in that it isn’t directly used during the function. The return address is a parameter which tells the function. where to resume executing after the function is completed. This is needed because functions can be called to do processing from many different parts of your program, and the function needs to be able to get back to wherever it was called from. In most programming languages, this parameter is passed automatically when the function is called. In assembly language, the call instruction handles passing the return address for you, and ret handles using that address to return back to where you called the function from.

      Return value. The return value is the main method of transferring data back to the main program. Most programming languages only allow a single return value for a function.
