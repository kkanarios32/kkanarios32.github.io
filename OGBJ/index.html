What happens during a function call [OGBJ]

  • July 8, 2026
  • Kellen Kanarios

For this note, we will assume we are using the C calling convention. More details in [bartlettProgrammingGround].

The first thing we do is push the arguments onto the stack (in reverse order). Then execute a call instruction. The call instruction does two things:

  1. It pushes the return address onto the stack. This is the address of the next instruction after the function call.
  2. It then modifies eip to point to the function to be called.

Now we have jumped to the function to be called. However, there is still some bookkeeping that must be done within a function. Namely, we must keep track of where the stack started on a function call. To do so, we copy esp (stack pointer) into ebp (base register). This allows us to access each parameter stored to the stack by the caller via \(N*4 + 4\) ebp.

Finally, we simply allocate space on the stack by subtracting the size of the arguments from esp. This is the new stack pointer.

Now we have to return from the function. To do so, we first store the return value in the special register eax. Then, we must return to where we left off prior to calling the function. Recall that we saved the next instruction address at the top of the stack when we first called the function. There are now other local variables, etc. in the stack so we have to unwind it back. We can do this simply by 

movl %ebp, %esp
popl %ebp
ret
Here, the ret instruction just pops whatever is on the top of the stack and sets rip to it.

We are now back in the calling code with the return value stored in eax. However, remember that the caller pushed the function parameters onto the stack. Therefore, we must also pop all of these parameters off before continuing execution.
