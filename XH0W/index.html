Scalable Architectures for Robot Learning [XH0W]

  • September 15, 2025
  • Kellen Kanarios

We have the following taxonomy for robot learning:

  1. Autonomous driving: very perception heavy problem -> Learning for perception
  2. Locomotion: very controls-heavy problem -> Learning for control
  3. Manipulation: perception + controls -> End-to-end visuomotor policy learning

Specialist models are brittle i.e. sensitive to differences in lighting or camera angle.

More data -> generalist models

1. Robotics as multi-modal sequence modeling

  • September 15, 2025
  • Kellen Kanarios

Predict "action chunks" i.e. a sequence of actions (roughly 1sec into the future)

Use imitation learning: \[\max _{\theta } \sum _{(a,o,T)} \pi _{\theta }(a \mid o, T)\] where \(T\) is the task identifier.

Datasets: DROID, Open-X embodiment.

Vision-Language-Action model: given images, text description output actions dx, dy, dz.

  1. Scalable training
  2. Flexible inputs + outpus
  3. Internet scale pre-training

How do we output actions? Discretization via uniform bins.

OpenVLA:

  • Two stream encoder for spatial (DinoV2) and semantic (CLIP) features.

2. Robotics-specific modeling considerations

  • September 15, 2025
  • Kellen Kanarios

Two main considerations

  1. Action tokenization
  2. Speed

OpenVLA models degrade as you increase the control frequency.

  • Training in naive way, resort to "token-copying".

Idea: compress actions. Discrete cosine transform.

FAST:

  1. Apply DCT
  2. Quantize
  3. Flatten
  4. BPE

Autoregressive decoding is slow AF:

  • Use sepcialized decoding head.
  • Diffusion to get the whole chunk at once.
    • 10 vs. 60 autoregressive steps
    • 300m vs 3b params during inference.
    • Model trains 5x slower

Train and inference via doing BOTH at the same time.

  • Freeze action expert gradients to VLM.
  • Train VLM via autoregressive predictions.
  • Knowledge insulating VLAs
