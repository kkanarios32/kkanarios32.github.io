Distributed Shared Memory (GPU) [OGCS8]

  • June 1, 2026
  • Kellen Kanarios

Because the CTAs in a cluster live on neighboring SMs connected by a dedicated SM-to-SM network, the shared memory of all CTAs in the cluster is exposed as a single logical address space. A thread in one CTA can read/write/atomic into another CTA's shared memory directly, without round-tripping through global memory (L2/DRAM).

cooperative_group.cu
        #include <cooperative_groups.h>
namespace cg = cooperative_groups;

__global__ void __cluster_dims__(2,1,1) kernel(...) {
    cg::cluster_group cluster = cg::this_cluster();
    extern __shared__ int smem[];

    // Get a pointer to another CTA's shared memory
    unsigned int rank = cluster.block_rank();      // my CTA's id in cluster
    int *remote = cluster.map_shared_rank(smem, /*target rank=*/ rank ^ 1);

    // Now read/write the neighbor's shared memory
    remote[threadIdx.x] = threadIdx.x;

    cluster.sync();   // cluster-wide barrier
}

      Cooperative Thread Array (CTA) [W2D5]

      • June 1, 2026

      NVIDIA's hardware-level term for what CUDA programmers call a thread block.

      A CTA is a group of threads that execute on the same Streaming Multiprocessor (SM), can synchronize via barriers (__syncthreads()), and share fast on-chip shared memory (Shared memory (GPU)). Threads within a CTA are scheduled and executed in groups of 32 called warps.
