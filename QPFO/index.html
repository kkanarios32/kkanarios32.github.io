Data parallelism [QPFO]

  • May 22, 2026
  • Kellen Kanarios

A form of parallelism where the model is replicated across multiple devices and performed on different data.

  • Data sharding: data is split across multiple devices i.e. 32 samples and 8 devices results in 4 unique samples per device.
  • Parameter replication: on each device, we need to replcicate the model parameters to run the forward pass for the unique samples.
  • Gradient aggregation: backward pass we need to perform an all-reduce to sum and average the gradients across devices. This is automatically done by @nnx.jit.
