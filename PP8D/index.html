Definition.  constexpr [PP8D]

  • June 1, 2026

A hint to the compiler in C++ that the value or function can be evaluated at compile time. This allows the compiler to pre-compute the function / value and hardcode it into the resulting binary, eliminating any instructions involved in the computation from the final binary.

Idea. Shift work from every run on every user's machine to once on the build machine.

Remark. A constexpr function is not forced to run at compile time. It is only done only when the arguments are const.

    Struct (C++) [CCIB]

    • June 1, 2026

    A struct is a class whose members are public by default. The two are the same mechanism — struct simply flips the default access — so the choice is a convention: use struct for passive aggregates of data with no invariant to maintain, and class when access control is enforcing an invariant.

    Union (C++) [W375]

    • June 1, 2026

    A union is a struct in which all members are allocated at the same address. This ensures that the union occupies only as much space as its largest member.

    A union can hold a value for only one member at a time.

    Problem. You can only read the active member of a union. 
    union U { int i; float f; };
U u;
u.i = 42;
float x = u.f;  // UB — reading the inactive member
    Problem. No automatic lifetime management for non-trivial types. If a union holds an object with a non-trivial constructor/destructor (like std::string), the compiler won't call them for you. You must manually placement-new and explicitly call the destructor, and getting this wrong causes leaks or corruption. 
    union U { std::string s; int i; };  // destructor not called automatically
    Remark. Avoid "naked" unions; opt for "tagged" unions. 
    struct Value {
    enum class Type { Int, Float, Str } tag;
    union {
        int   i;
        float f;
        char  s[32];
    };
};
    This along with lifetime management is the basis of std::variant.

    Enumerations (C++) [QK4C]

    • June 2, 2026
    • Kellen Kanarios

    User-defined type in C++ that is a set of named values designed to make code more readable and less error prone.

    Example. 
    enum class Color { red, green, blue };

Color c = Color::red;

    If you want enumerator values to be int, you can use enum.

    Example. 
    enum Color { red, green, blue };

int color = green     // color = 1
    Note that this is an error if Color is enum class.
    Remark. Use enumerations to represent sets of named constants
    Remark. Prefer enum class to enum when possible

    Concrete and abstract types

    • June 1, 2026
    • Kellen Kanarios

    The pivotal design axis is whether a type's representation is part of its definition. A concrete class commits to its layout, so it can live on the stack and be copied by value; an abstract type hides representation entirely behind pure virtual functions, which forces heap allocation and pointer-based use. The container is the running example that both build on.

    Definition. Concrete Class (C++) [OGCSC]

    • June 5, 2026
    • Kellen Kanarios

    A class that behaves "just like built-in types". The defining characteristic of a concrete type is that its representation is part of its definition.

    Because the representation is fixed you can

    1. Allocate objects directly on the stack.
    2. Copy and assign them by value
    3. Have them as direct members of other classes.
    Example. complex, vector, and string are concrete classes.

    Definition. Virtual Function Table (C++) [OGCSH]

    • June 9, 2026
    • Kellen Kanarios

    Each class with a virtual function gets a virtual table: a static, per-class, array of function pointers with one entry per virtual function.

    The virtual table is created by the compiler and is stored in the object's memory.

    vtable layout:
  [ offset-to-top  ]   // slot -2
  [ type_info*     ]   // slot -1  ← RTTI
  [ &virtual_fn_0  ]   // slot  0
  [ &virtual_fn_1  ]   // slot  1
  ...
    Remark. The type_info pointer is needed to support runtime type-checking / dynamic casting .

    During construction, the vptr is initialized to point to the base class's vtable. In each classes vtable, there is a thunk (offset-to-top) that tells

    Example. 
    struct A {
    virtual void f();
    int a;
};

struct B {
    virtual void g();
    int b;
};

struct D : A, B {
    void g() override;   // overrides B::g
    int d;
};
    offset 0:   [ vptr_A ]   ← D's primary vtable (A subobject)
offset 8:   [ a      ]
offset 16:  [ vptr_B ]   ← D's secondary vtable (B subobject)
offset 24:  [ b      ]
offset 32:  [ d      ]

    Every object of a class with a virtual function carries a hidden vptr (virtual pointer) that points to the class's virtual table.

    Example. Consider the following class hierarchy: 
    struct Base {
    virtual void f();
    virtual void g();
    int x;
};

struct Derived : Base {
    void f() override;   // overrides Base::f
    void h();            // non-virtual, not in vtable
    int y;
};
    This yields the following components for each type of object: 
    Base object:        Derived object:
[ vptr ]            [ vptr ]      → points to Derived's vtable
[ x    ]            [ x    ]
                    [ y    ]

    auto keyword [JXI3]

    • June 1, 2026

    auto is automatic type deduction in C++.

    In Generic Programming, auto is crucial because types may not be known.

    Question. When should you NOT use auto?
    Answer. In C++ they provide two reasons:
    1. When you want to make the type visible clearly to the readers of your code.
    2. You want to be explicit on the variable range or precision i.e. auto x = 1; will be an int. If you want to enforce a double, then you must specify so explicitly.
    Remark. auto x{v}; deduces single value's type in modern C++, but in C++11/14 it deduces std::initializer_list. auto x = {v}; remains an std::initializer_list in modern C++.

    Exceptions (error handling) [D770]

    • June 3, 2026

    In C++, exceptions are thrown via the throw keyword.

    The throw keyword transfers control to exception handler. This then proceeds as

    1. Allocate heap memory for the exception object.
    2. Invoke the unwinder.
      1. Walk the stack frame-by-frame to checking to see if any frame has a catch with matching type.
      2. Walks the stack again, restoring registers / SP per frame.
        • Cleaning up all the invalid stack frames between the throw and catch.
      3. Transfers control to the catch block.

    If no matching catch block is found, the program calls std::terminate

    You can annotate a function with noexcept to indicate that it never throws exceptions. This is mainly used as an annotation to enable better choices by the compiler

    Example. The compiler may call copy constructor instead of move constructor if the move constructer is not marked noexcept; If an exception were thrown during moving, then the object would be in a broken state.

        struct A {
    virtual void f();
    int a;
};

struct B {
    virtual void g();
    int b;
};

struct D : A, B {
    void g() override;   // overrides B::g
    int d;
};
        offset 0:   [ vptr_A ]   ← D's primary vtable (A subobject)
offset 8:   [ a      ]
offset 16:  [ vptr_B ]   ← D's secondary vtable (B subobject)
offset 24:  [ b      ]
offset 32:  [ d      ]

        Every object of a class with a virtual function carries a hidden vptr (virtual pointer) that points to the class's virtual table.

        Example. Consider the following class hierarchy: 
        struct Base {
    virtual void f();
    virtual void g();
    int x;
};

struct Derived : Base {
    void f() override;   // overrides Base::f
    void h();            // non-virtual, not in vtable
    int y;
};
        This yields the following components for each type of object: 
        Base object:        Derived object:
[ vptr ]            [ vptr ]      → points to Derived's vtable
[ x    ]            [ x    ]
                    [ y    ]

        Definition. constexpr [PP8D]

        • June 1, 2026

        A hint to the compiler in C++ that the value or function can be evaluated at compile time. This allows the compiler to pre-compute the function / value and hardcode it into the resulting binary, eliminating any instructions involved in the computation from the final binary.

        Idea. Shift work from every run on every user's machine to once on the build machine.

        Remark. A constexpr function is not forced to run at compile time. It is only done only when the arguments are const.

        auto keyword [JXI3]

        • June 1, 2026

        auto is automatic type deduction in C++.

        In Generic Programming, auto is crucial because types may not be known.

        Question. When should you NOT use auto?
        Answer. In C++ they provide two reasons:
        1. When you want to make the type visible clearly to the readers of your code.
        2. You want to be explicit on the variable range or precision i.e. auto x = 1; will be an int. If you want to enforce a double, then you must specify so explicitly.
        Remark. auto x{v}; deduces single value's type in modern C++, but in C++11/14 it deduces std::initializer_list. auto x = {v}; remains an std::initializer_list in modern C++.

        Exceptions (error handling) [D770]

        • June 3, 2026

        In C++, exceptions are thrown via the throw keyword.

        The throw keyword transfers control to exception handler. This then proceeds as

        1. Allocate heap memory for the exception object.
        2. Invoke the unwinder.
          1. Walk the stack frame-by-frame to checking to see if any frame has a catch with matching type.
          2. Walks the stack again, restoring registers / SP per frame.
            • Cleaning up all the invalid stack frames between the throw and catch.
          3. Transfers control to the catch block.

        If no matching catch block is found, the program calls std::terminate

        You can annotate a function with noexcept to indicate that it never throws exceptions. This is mainly used as an annotation to enable better choices by the compiler

        Example. The compiler may call copy constructor instead of move constructor if the move constructer is not marked noexcept; If an exception were thrown during moving, then the object would be in a broken state.
