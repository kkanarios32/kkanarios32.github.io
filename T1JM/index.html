git bisect [T1JM]

  • June 17, 2026

Useful tool for finding a breaking commit. Writeup taken from here.

Use two terminals. One is used for the bisection process and the other is used to run the program to tell whether the current commit is good or bad.

  1. Start with git bisect start to initialize the bisection process.
  2. Find a good commit and mark it with git bisect good <commit>.
  3. Run git bisect bad <commit> to mark the bad commit.
  4. Repeat until you find the breaking commit.
  5. Use git bisect reset to restore to the state you started from.
Remark. Can use git bisect skip to skip a specific commit, or git bisect visualize --oneline to see current progress.
