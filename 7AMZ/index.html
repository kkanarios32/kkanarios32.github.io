1. Benchmarking Humanoid Intelligence

  • September 17, 2025
  • Kellen Kanarios

Locomotion tasks: can humanoids move around? Vary terrain, avoid obstacles.

Loco-manipulation tasks: can humanoids interact with the environment?

HumanoidBench

Other manipulation / locomotion benchmarks are too trivial to see drastic performance between algorithms.

Learning algorithms struggle with large action space

Action space - number of degrees of freedom. Humanoid w/o hands - 19dof, w/ hands 61dof.

Idea: decouple dof. Problem want whole body control.

Idea: hierarchical learning via domain knowledge. learn to position hands. Extended to library of skills in Skillblender.

2. Scaling sim-to-real learning for humanoids

  • September 17, 2025
  • Kellen Kanarios

Robot / environment:

  • identify what you can (system identification).
  • what you cannot identify randomize!

Reward

  • stay away from things you cannot identify.

Simulator: training on cpu takes forever (enter IsaacGym, Mujoco Playground).

Mujoco Playground limitations

  • JIT compilation can be slow
  • Computation time scales with the number of potential contacts
  • MJX warp is now supported (100x speedup)

3. Where can we do better?

  • September 17, 2025
  • Kellen Kanarios

Fast TD3

FastTD3 vs. TD3

  1. Massively parallel environments
  2. Large-batch training
  3. Distributional RL
    • Discretize value space and output distribution over values
    • Trained with cross entropy loss
Body Tranformer,
  1. Inductive bias
  2. Computationally efficient
  3. Multi-frequency reasoning
  4. Localized credit assignment

Idea: construct graph to connect certain parts of body via embodiment graph. Have attention layers that only attend to neighbors on the graph via the mask being a graph mask.

Mask is sparse, allowing much more computationally efficient attention computation.
