Warp divergence serializes both sides of a branch [OGCSM]

  • July 6, 2026
  • Kellen Kanarios

When threads in a warp take different branches of a conditional, the warp does not split into smaller warps. Instead, the active mask selects which lanes execute each branch; lanes on the inactive side run dummy instructions until the warp reconverges at the post-conditional join point. The programmer writes ordinary scalar code with if/else, and the hardware silently masks the inactive lanes.

This is the chief ergonomic win over SIMD: the compiler does not have to prove that all lanes will take the same path. The cost is that divergent code pays for both sides — a 32-wide warp where 16 threads take the if-branch and 16 take the else-branch executes both branches sequentially with half the lanes idle each time.

The hardware sequences the two paths with a reconvergence stack — or, on recent GPUs, a barrier-based equivalent that avoids its deadlocks. The standard software mitigation is JKNL: group threads with the same control-flow trajectory into one warp so the branch never diverges in the first place.

Context

    Definition. SIMT [7F19]

    • May 13, 2026
    • Kellen Kanarios

    Single instruction, multiple threads. A GPU streaming multiprocessor issues one instruction per cycle to a group of threads — a warp — but each thread within the group keeps its own program counter and register state. The hardware enforces lockstep execution within the warp while preserving the illusion of independent threads to the programmer.

    SIMT is the GPU's answer to a question the Vector ISA / SIMD world struggled with: how do you get the throughput of wide vector execution without forcing the programmer to manually vectorize? Two mechanisms make this work.

    Grouping flexibility

    • May 13, 2026
    • Kellen Kanarios

    Threads are assigned to warps based on warp-ID, but the hardware and compiler are free to regroup. JKNL re-bundles threads with similar control-flow trajectories into the same warp so that future branches stay non-divergent. This is a class of optimization that pure SIMD cannot express — a SIMD ISA fixes the lane assignment at compile time.

    The hardware cost of SIMT relative to SIMD is the per-thread PC and register state. The benefit is that a single binary runs efficiently across a wide range of dynamic control flow, which is why the model has dominated GPU programming since CUDA. The scoreboard mechanism described in GPU register scoreboard operates per-warp rather than per-thread precisely because SIMT treats the warp as the unit of issue.

    Remark. SIMT is sometimes described as "SIMD with a per-lane PC," which captures the implementation but understates the programming-model impact: vectorization becomes a runtime, not a compile-time, decision.

      A reconvergence stack sequences a warp's divergent paths and can deadlock [KY0M]

      • July 11, 2026
      • Kellen Kanarios

      When a warp diverges, the hardware needs to remember where the split paths rejoin. The classical mechanism is a reconvergence stack: each entry holds a next-PC, a reconvergence PC (the post-conditional join point), and an active mask. On a divergent branch the hardware pushes both sides and executes them one at a time, popping back to the reconvergence point as each side completes; entries are pushed largest-mask-first to bound stack depth.

      The stack has a fatal flaw. A thread that exits a loop diverges and is stacked below threads still spinning inside it — so if those spinning threads wait on a lock the exited thread already holds, neither can proceed and the program deadlocks. This is why modern NVIDIA hardware abandons the stack for a barrier-based scheme: each divergence point is tracked by a 32-bit participation mask, letting the scheduler switch between branches arbitrarily and guarantee independent thread progress.

      Warp [X9VZ]

      • May 12, 2026

      A warp is the unit of scheduling on an NVIDIA GPU: a group of ~32 threads that issue one instruction per cycle in lockstep. Each functional unit in the SM has as many lanes as a warp has threads, so a single fetched instruction fans out across all 32 lanes at once. The warp — not the thread — is what the hardware fetches, schedules, and tracks; the per-thread SIMT programming model is an illusion built on top of it.

      Because the warp is the atom of execution, the two facts that dominate GPU performance are both properties of the warp: it hides memory latency by switching among resident warps instead of stalling, and it serializes both sides of a branch when its threads diverge.
