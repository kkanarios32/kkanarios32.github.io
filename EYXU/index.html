MSI protocol [EYXU]

  • May 12, 2026

The minimal Cache coherence state machine that supports write-back caches. Each cache line is in one of three states:

  1. Invalid (I) — this cache does not hold a valid copy.
  2. Shared (S) — clean copy, read permission, possibly held by other caches.
  3. Modified (M) — only copy, exclusive read+write permission, dirty (must be written back on eviction).

Bus messages used:

  • BusRd — request a shared copy.
  • BusRdX — request exclusive ownership for a write.
  • BusInv — invalidate other caches on a write hit.
  • BusWB — write back a dirty block on eviction.
  • BusReply — supply the value to a requester instead of memory.
Remark. S → M requires either a BusInv or BusRdX so that other sharers invalidate. The move from S to M is not silent.
Remark. MESI protocol adds a fourth state (Exclusive) to optimize the common case of thread-private data — a line read by exactly one core can be upgraded to writable without a bus transaction.

    MESI protocol [30XT]

    • May 12, 2026

    MSI protocol augmented with an Exclusive state. The four states:

    1. Modified (M) — only copy, dirty, writable.
    2. Exclusive (E) — only copy, clean, writable.
    3. Shared (S) — clean, possibly held by other caches, read-only.
    4. Invalid (I) — no valid copy.

    The crucial addition is that a line transitioning S → M needs a bus transaction (other sharers must invalidate), but a line transitioning E → M is silent: nobody else has a copy, so nobody needs to be told. Eviction from E also skips the writeback (E is clean), saving memory traffic.

    MESI is dramatically better than MSI for thread-private data: a thread that reads then writes its private variable goes I → E → M with only one bus transaction, instead of I → S → M with two.

    The next refinement is MOESI protocol, which adds an Owned state for shared-but-dirty lines.

    Cache coherence [GA7N]

    • May 12, 2026

    In a Multicore system where every core has its own Cache hierarchy, the same memory address can live in multiple caches simultaneously. A cache coherence protocol is the mechanism that keeps those copies consistent — propagating writes, invalidating stale copies, and preserving two invariants:

    1. Single-writer invariant: at any moment, at most one core has permission to write a given line. No other core can read or write it concurrently.
    2. Data-value invariant: all cores see the same order of updates to a given line.

    Two organizational families:

    • snoop-based coherence — every cache watches a shared bus and reacts to other cores' traffic. Scales to ~8 cores.
    • directory-based coherence — a structure tracks who has a copy of each line; coherence messages are point-to-point. Scales to hundreds of cores at the cost of an extra hop.

    The concrete state machines are protocols like MSI protocol, MESI protocol, MOESI protocol, and DEC Firefly, each adding states to handle more cases or to optimize specific sharing patterns. A worked implementation of directory-based MESI in Murphi lives in Implementing Directory-based MESI in Murphi.
